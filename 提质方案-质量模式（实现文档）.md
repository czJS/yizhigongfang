## 1. 目标与边界
**目标**：在全本地条件下，把质量模式的字幕/翻译/配音提升到“接近线上高质量”的稳定水平，并具备可审校、可回归、可运营的工程能力。

**边界**
- 允许使用本地 LLM，但必须可配置、可回退、可诊断（避免随机失败）。
- 不以提速为目标，但必须保证**输出稳定与可交付**（不截短、不丢段、不污染格式）。

---

## 2. 质量模式提质的核心抓手（来自成熟实践）
参考 `VideoLingo` 等成熟流水线，质量提升主要来自三类工程化能力：
- **字幕形态标准化**：Netflix 化（单行优先、可读性控制、智能拆句）。
- **翻译流程多步化**：忠实→反思→改写（Translate-Reflect-Adaptation），并用严格格式校验保证稳定。
- **配音任务化**：把字幕稿变成可朗读的 TTS 稿，按估时/节奏对齐，避免“飙语速/吞字/奇怪停顿”。

---

## 3. 实现清单（质量模式专属）

### 3.0 P0 已落地能力（可验证）
本项目已落地一批 P0 的“稳定提质”能力（默认在质量模式 `config/quality.yaml` 里开启；也可在前端参数里显式开关），并且每一项都能通过产物/报告验证：

**(A) 字幕回流/整形（不改段数，兼容审校 1:1 行号）**
- **做什么**：
  - 英文字幕“单行净化”（去掉多余换行/空白）。
  - 可选：对超长英文做软换行（降低“最长行超限”告警）。
  - 可选：阅读速度整形（CPS fix）：当 CPS 超限时，把该段的 **end 时间向后延伸到空隙里**（不与下一段重叠），以降低 CPS 超限告警。
- **对应参数（前端/后端一致）**：
  - `subtitle_postprocess_enable`
  - `subtitle_wrap_enable`、`subtitle_wrap_max_lines`
  - `subtitle_cps_fix_enable`、`subtitle_cps_safety_gap`
- **阈值来源**：与 `quality_gates.max_cps / max_chars_per_line` 同源，避免“看不见的规则”。

**(B) “字幕稿”与 “TTS 稿”分离（最小版）**
- **做什么**：生成 `eng_tts.srt`（朗读稿），并优先用其作为 TTS 输入；字幕稿仍为 `eng.srt`。
- **对应参数**：`tts_script_enable`
- **验收目标**：TTS 不读垃圾符号/括号旁白，停顿更自然，且字幕观感不被“朗读优化”牵连。

**(C) 可验证方式（验收用这三条即可）**
- **产物**：任务目录下出现 `eng_tts.srt`（开启 `tts_script_enable` 时）。
- **日志**：`run.log` 中出现 `[p0] subtitle_wrap...` / `[p0] subtitle_cps_fix...` 统计行（开启对应能力时）。
- **质量报告**：`quality_report.json` 中 `line_length.hits_n`、`reading_speed.hits_n` 可对比开启前后是否下降（阈值来自 `quality_gates`）。

### 3.1 多步翻译（faithful → reflect → adapt）
**做什么**
- 第一步：逐行忠实直译（信息不丢）
- 第二步：逐行指出问题（术语、歧义、风格、口语化）
- 第三步：产出最终可读字幕（更自然、符合目标语言表达）

**关键要求（工程化）**
- 每步都必须“一行对应一行”，输出只允许英文，不得出现编号/解释/引用。
- 若输出结构不合格（行数不对、含中文、出现编号）：按策略重试/回退到更保守 prompt。
- 与术语表联动：必须使用指定译名（输出后再校验）。

**验收**
- Golden Set 上：语义一致性提升、格式污染为 0、术语一致性达标。

### 3.2 字幕产品规范化（Netflix 化）
**做什么**
- 单行优先（必要时双行），控制每行长度与阅读速度。
- 长句智能拆分：标点优先，其次依存/连接词，再硬切兜底。
- 时间轴微调：减少闪烁与过短段（不改变语义前提下）。

**验收**
- 超长行比例、阅读速度超限比例低于阈值。
- 字幕观感稳定（无一屏塞满、无频繁闪烁）。

### 3.3 “字幕稿”与 “TTS 稿”分离
**做什么**
- 字幕稿：信息完整、适合阅读。
- TTS 稿：适合朗读（去脚注/括号/奇怪标记；补必要停顿；避免难读符号）。

**验收**
- TTS 不读出垃圾字符（如 `.`、引用标记等）。
- 配音节奏自然，不依赖极端倍速压缩。

### 3.4 配音任务化（估时 → 裁剪/扩写 → 对齐）
**做什么**
- 为每条字幕估算朗读时长（规则/统计模型即可，不必 LLM）。
- 超时则裁剪（保持要点），不足则保持自然停顿（补静音而非堆词）。
- 对齐策略优先“补静音保持时间轴”，少用强加速（避免怪声）。

**验收**
- `tts_full.wav` 覆盖范围与源视频一致（不截短）。
- 语速超限比例在阈值内。

### 3.5 质量模式稳定性与可诊断（必须）
**做什么**
- 输出结构校验（英文纯净、行数对齐、无编号/解释）。
- 失败日志必须包含：模型名、关键参数、失败阶段、可复现输入。
- 允许显式回退策略（例如从“多步”回退到“单步忠实”），但需用户可见。

---

## 4. 质量模式验收指标（建议）
对照 `验收清单（质量与交付）.md`，补充质量模式“提质指标”：
- **语义一致性**：对 Golden Set 的关键事实覆盖率
- **自然度**：人工抽检评分（可按 1~5）
- **术语一致性**：漂移率/多译率
- **字幕可读性**：长度/阅读速度超限比例
- **格式污染**：编号/解释/中文残留为 0

---

## 5. 里程碑建议（质量模式）
- **Q1（可交付稳定）**：结构校验 + 术语强约束 + 不截短保证 + 基础字幕规范
- **Q2（接近线上观感）**：多步翻译 + 智能拆句 + TTS 稿分离
- **Q3（可运营）**：审校工作流 + TM 回写 + Golden Set 回归报表自动化
