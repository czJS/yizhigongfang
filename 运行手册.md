# YizhiStudio 本地运行与常见问题速查

## 后端（Docker）
- 镜像构建（确保 `bin/` 下是 **Linux x86_64** 可执行文件）：
  - `cd /Users/chengzheng/Desktop/译制工坊`
  - `docker build -t yizhi-backend:dev .`
- 运行容器（避免用宿主机的 macOS 可执行文件覆盖 `/app/bin`，仅挂载数据）：
  - `docker run -d --name yizhi-backend-dev -p 5175:5175 -v $(pwd)/outputs:/app/outputs -v $(pwd)/assets:/app/assets yizhi-backend:dev`
- 健康检查：`curl http://127.0.0.1:5175/api/health`

## 前端（Vite + Electron，开发态）
1) 关闭旧进程并清理端口：
```
cd /Users/chengzheng/Desktop/译制工坊/frontend
./node_modules/.bin/kill-port 5173 5174 || true
pkill -f "vite" || true
pkill -f "electron electron/main.js" || true
```
2) 启动 Vite（保持前台，待 ready）：
```
./node_modules/.bin/vite --host --port 5173 --clearScreen false
```
看到 `Local: http://localhost:5173/` 后再执行第 3 步。
3) 新终端启动 Electron（固定同一 host/port，禁用代理）：
```
cd /Users/chengzheng/Desktop/译制工坊/frontend
NO_PROXY=127.0.0.1,localhost VITE_DEV_SERVER_URL=http://localhost:5173 ./node_modules/.bin/electron electron/main.js
```

## 常见问题与处理
- 浏览器 504 / “Outdated Request”：
  - 原因：系统/代理软件劫持本地 127.0.0.1 流量或缓存过期。
  - 处理：关闭/绕过代理（直连 `127.0.0.1, localhost`），无痕窗口或 `--no-proxy-server` 启动浏览器，硬刷新。
- Electron 白屏：
  - 确认 Vite 已 ready（端口 5173 监听且能 `curl` 200）。
  - 确认 Electron 启动参数使用与 Vite 一致的 `http://localhost:5173`，并带 `NO_PROXY=127.0.0.1,localhost`。
- `OSError: [Errno 8] Exec format error: '/app/bin/main'`：
  - 原因：`/app/bin/main`（whisper.cpp）是 macOS/错误架构；容器需 Linux x86_64。
  - 解决：
    1. 下载 Linux x86_64 版 whisper.cpp 可执行文件（`main`），替换 `bin/main`，`chmod +x bin/main`。
    2. 同步检查 `bin/piper`、`bin/ffmpeg` 也为 Linux x86_64，`chmod +x`。
    3. 重新构建镜像：`docker build -t yizhi-backend:dev .`
    4. 运行容器时不要挂载宿主机的 `bin/`，否则会覆盖镜像内的 Linux 可执行文件。

## 快速启动清单
- 后端容器：`docker start yizhi-backend-dev`（如不存在则重新 build/run）。
- 前端：按“前端开发态”三步走，保持 host/port 一致，Electron 带 NO_PROXY。
