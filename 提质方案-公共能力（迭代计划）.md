## 4. 后续迭代（按优先级）

### 4.1 Iter-1（P0 增强）：门禁阈值配置化 + UI 对齐
**目标**：让“门禁阈值”从代码里消失，全部来自配置；前端实时标红与后端门禁一致，方便运营调参与回归。

- **交付物**
  - `config/defaults.yaml` 增加/完善 `quality_gates`（例如 max_cps/max_chars_per_line/max_duration_diff 等）
  - `/api/config` 返回的配置中包含 `quality_gates`，前端只读展示
  - 前端审校表格的 CPS/最长行阈值从 `quality_gates` 获取（不再写死）
- **验收**
  - 修改 `quality_gates` 后无需改代码即可影响门禁与前端红标
  - 门禁报告与审校实时红标在同一阈值下结果一致

### 4.2 Iter-2（P1）：句子单元（合并→翻译→映射）公共闭环
**目标**：把“ASR 片段”转为“可翻译句子单元”，并保存映射关系，便于审校与回归（两条模式共用）。

- **交付物**
  - `sentence_units.json`：句子单元列表、合并策略、unit ↔ segment 映射
  - 产物可视化（前端可下载/查看映射）
- **验收**
  - 时间轴单调，成片不截短
  - 相比不合并：超长行比例下降、明显跑偏降低（Golden Set 抽样对比）

### 4.3 Iter-3（P1）：TM（翻译记忆）最小可用 + 审校可选回写
**目标**：把“人工最后 10%”沉淀为资产，跨片复用，越用越好（仍离线可用）。

- **交付物**
  - TM 本地存储（JSONL/SQLite 二选一），支持 exact + 模糊检索（阈值可配）
  - 审校确认后可选回写 TM/Glossary（显式开关 + 变更预览）
- **验收**
  - 第二次处理同类视频时，术语/句式一致性显著提升
  - 回写可追溯（记录来源任务/时间/操作者）

### 4.4 Iter-4（P2）：回归报告自动生成与归档
**目标**：Golden Set 不只是“文件夹”，而是每次迭代都能自动输出对比报告，避免退化。

- **交付物**
  - 回归脚本：对比 `quality_report.json` + 关键产物差异（字幕 diff、指标对比）
  - 归档目录与命名规范（按日期/版本/commit）
- **验收**
  - 每次发版前后能自动生成可读对比报告并归档
