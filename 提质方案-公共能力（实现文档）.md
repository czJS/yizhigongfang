## 1. 文档目的
将“轻量模式/质量模式”的提质工作抽象成**可复用的公共能力**，用于产品化落地、排期与验收。

本文件只描述**两条模式共用**的实现内容；模式差异部分见：
- `提质方案-轻量模式（实现文档）.md`
- `提质方案-质量模式（实现文档）.md`

相关上位文档：
- `产品需求文档（PRD）.md`
- `验收清单（质量与交付）.md`

---

## 2. 适用范围与原则
- **离线优先**：任何能力都必须支持离线模式（不依赖外网）。
- **质量可控**：输出必须可解释、可复现、可回归；避免“看模型心情”。
- **不隐式降级**：任何失败不得偷偷切换到另一模式/另一后端（除非用户显式选择）。

---

## 3. 公共能力清单（必须做）

### 3.1 术语体系（Glossary / TM / 一致性）
**目标**：让译名/专名/固定表达在全片与跨片一致，降低“漂移”。

**交付物**
- Glossary（术语表）文件：建议 `assets/quality/glossary/`（路径仅建议，具体可调整）
  - 字段建议：`src`（中文）、`tgt`（英文）、`aliases`、`forbidden`、`note`、`scope`（全局/项目/视频）
- TM（翻译记忆）与复用策略：本地可检索（模糊匹配阈值可配置）
- 一致性检查报告：漂移项列表（出现位置、建议替换）

**验收要点**
- 同一术语在同一任务内不得出现多译（除非写入别名白名单）。
- 输出中术语覆盖率达到阈值（按内容类型配置）。

### 3.2 文本结构化：断句/合并/再映射
**目标**：把“ASR 时间片段”转为“可翻译句子单元”，再映射回时间轴，减少碎片化导致的语义跑偏。

**交付物**
- 断句策略（规则优先，NLP 可选）：标点/停顿/VAD/长度阈值
- 映射策略：句子单元 ↔ 原时间轴片段（可追溯）
- 产物：保存“句子单元”中间结果（便于审校与回归）

**验收要点**
- 映射后时间轴保持单调且覆盖范围合理（不得导致成片截短）。
- 断句后字幕可读性提升（超长行比例下降）。

### 3.3 质量门禁（自动检查）
**目标**：用不依赖大模型的规则把“明显不可用”挡在流水线之外，保证可交付底线。

**门禁项（建议最小集）**
- 英文纯净：无中文/全角符号残留（专名白名单例外）
- 行数/结构：输入输出行数对齐（或有明确对齐策略），空行率不超阈值
- 可读性：单行长度阈值、阅读速度 CPS/WPM 阈值
- 异常检测：重复 n-gram、明显模板化/无关句式爆发
- 术语一致性：漂移项为 0 或在白名单内

**交付物**
- 每任务生成 `quality_report.json`（或 md）：
  - 门禁通过/失败原因
  - 关键指标数值
  - 漂移项与建议替换

### 3.4 金标准与回归（Golden Set）
**目标**：版本迭代可回归，避免“修一个坏两个”。

**交付物**
- Golden Set 素材清单（可逐步扩充，不必一次到位）
- 标准字幕：例如你已建立 `outputs/_golden_standard/chs.srt`、`outputs/_golden_standard/eng.srt`
- 回归报告模板：对比指标 + 典型差异片段 + 退化项列表

**验收要点**
- 每次发版前后可输出对比报告并归档。

### 3.5 审校工作流（商业化必备）
**目标**：让“模型输出”变成“可控交付”，把最后 10% 人工校对变成产品能力。

**交付物**
- 审校入口：允许用户只改 `eng.srt`（或句子单元）并触发“仅合成/仅封装”。
- 变更记录：审校前后 diff，可回写 TM/Glossary（可选）。

---

## 4. 里程碑建议（公共部分）
- **M1（底线可交付）**：质量门禁 + Golden Set + 基础术语表
- **M2（质量可控）**：TM + 一致性报告 + 审校入口
- **M3（规模化）**：多项目术语/记忆库 + 版本回归体系完善

---

## 5. 风险与对策
- **规则过严误伤**：门禁分“硬失败/软警告”，避免阻塞全部任务。
- **术语表维护成本**：支持导入/导出（xlsx/csv/json）与增量更新。
- **不同内容类型差异**：将阈值与风格模板按“内容类型”分档配置。
