# 打包与分发指南（Electron + Flask 后端）

## 结构
- 后端：`backend/app.py`（Flask），`backend/task_manager.py`。
- 前端：`frontend/`（React + Ant Design + Electron）。
- 资源：`bin/`（ffmpeg/whisper.cpp）、`assets/models/`（ASR ggml、MT、Coqui TTS）。
- 配置：`config/defaults.yaml`（路径、预设、UI 轮询等）。

## 本地开发
```bash
# 后端
cd /Users/chengzheng/Desktop/译制工坊
python -m venv .venv && source .venv/bin/activate
pip install -r backend/requirements.txt
python -m backend.app  # 默认 http://127.0.0.1:5175

# 前端
cd frontend
npm install
npm run dev   # 会启动 Vite + Electron（DEV URL 注入）
```

## 打包（mac .app/.dmg, Windows NSIS）
在 `frontend/electron-builder.yml` 已配置：
- `extraResources` 将 `bin/`、`assets/models/`、`assets/` 复制到应用资源目录：
  - macOS：`<App>.app/Contents/Resources/{bin,models}`
  - Windows：`<install_dir>/{bin,models}`
- 将 `backend/**` 和 `scripts/asr_translate_tts.py` 一并打入，离线可跑。
- NSIS 关闭 oneClick，允许用户目录安装，满足无管理员权限。

命令：
```bash
cd frontend
npm install
npm run dist   # 等价于 vite build + electron-builder
```

生成物：
- macOS：`dist_electron/YizhiStudio-<ver>-mac.dmg`（内含 `.app`，拖入 Applications 即可）。
- Windows：`dist_electron/YizhiStudio-<ver>-win.exe`（NSIS 向导，可选安装路径）。

## 运行时路径约定
- 模型：`models/asr/*.bin`，`models/mt/`，`models/tts/coqui/`。前端参数面板支持传入绝对/相对路径。
- 可执行：`bin/ffmpeg`，`bin/main`（whisper.cpp）。后端启动时自动注入。
- 输出与缓存：默认 `outputs/<task_id>/`；可通过 `config/defaults.yaml` 修改 `outputs_root`。
- 环境变量：`TTS_HOME` 指向 Coqui 缓存目录（已在后端 TaskManager 注入）；如需离线镜像，可设置 `HF_ENDPOINT`。

## 硬件与模型推荐
- 普通：`ggml-small-q5_0` + `Marian` + `Coqui CPU`，分句上限 50，VAD 关。
- 中端：`ggml-medium-q5_0` + `Marian` + `Coqui GPU`，分句 50，VAD 开。
- 高端：`ggml-medium-q5_0/large-v3 q5` + `NLLB-600M` + `Coqui GPU/VITS`，分句 40–50，VAD 开。

## 签名与权限
- macOS：建议 codesign + notarize，确保 `bin/` 动态库（如 piper/espeak）通过 Gatekeeper。
- Windows：建议代码签名减少 SmartScreen 拦截；NSIS 默认用户目录安装。
