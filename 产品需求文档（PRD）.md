## 1. 背景与目标

### 1.1 背景
译制工坊是一套面向个人与小团队的“本地离线视频译制/配音”工具链，支持将中文视频自动生成英文字幕与英文配音并输出成品视频。

当前产品的核心竞争力是：
- **本地化**：无需依赖外网与云端服务即可完成全流程（在离线模式下仍可运行）。
- **分档能力**：提供 **轻量模式（lite）** 与 **质量模式（quality）** 两套流水线，适配不同硬件与质量诉求。

### 1.2 产品目标（商业化可落地）
- **稳定交付**：从“上传素材 → 生成成片”，流程可恢复、可重试、不会因单点失败导致整体崩溃。
- **质量可控**：提供“金标准”对照与自动化质量门禁，确保版本迭代不会显著退化。
- **离线优先**：默认离线可用；在线能力仅作为可选增强，且不影响离线主链路。
- **可运营**：对外可清晰说明两种模式的硬件门槛、能力差异与适用场景。

### 1.3 非目标
- 不以“实时”处理为目标（提速不是第一优先级）。
- 不承诺 100% 语义零偏差；提供“可审校、可修订”的工作流来达到商用质量。

---

## 2. 用户与场景

### 2.1 目标用户
- **个人创作者**：需要把中文短视频快速做成英文字幕/配音发布海外。
- **小团队译制**：需要批量处理，要求过程可追踪、产物可审校、质量可回归。
- **企业内部知识出海**：对隐私与离线要求高，不允许把素材上传云端。

### 2.2 典型场景
- 解说/科普/口播：语速快、信息密度高，需要字幕可读性与一致性控制。
- 剧情片段：人物/设定/专名多，需要术语表与一致性强约束。
- 课程录屏：长视频，要求断点续跑、批量任务、可审校。

## 3. 产品形态与模块

### 3.1 产品形态
- 桌面端/本地 Web GUI（前端）+ 本地后端（任务管理与流水线执行）。
- Docker 一键部署（面向非技术用户）与源码模式（面向开发/高级用户）。

### 3.2 核心模块
- **任务管理**：创建任务、查看进度、查看日志、重试/取消、产物下载。
- **素材管理**：上传/选择素材，校验是否为媒体文件，展示媒体信息（时长、分辨率、音轨）。
- **流水线执行器**：
  - 轻量模式：whisper.cpp ASR → Marian MT → Piper TTS → 合成
  - 质量模式：WhisperX ASR+对齐 → 本地 LLM 翻译/润色 → Coqui TTS → 合成
- **配置与预设**：模式对应的默认后端与模型路径由后端配置决定；前端字段只读、不可覆盖关键模型参数。
- **产物与中间产物**：音频、字幕、ASR/翻译 JSON、分段 TTS wav、最终视频。

## 4. 关键能力（“提质”导向的商业化方案）

### 4.1 质量可控体系（必须）
- **金标准（Golden Set）**：维护一组固定素材与对应标准字幕（中/英），用于回归测试与对比评分。
- **质量门禁（自动检查）**：不依赖大模型的硬规则检查：
  - 英文字幕纯英文（无中文/全角符号残留）
  - 行数/时轴对齐（无大量空行、无错位）
  - 可读性阈值（单行长度、阅读速度 CPS/WPM）
  - 术语一致性（同一术语不得漂移）
  - 异常模式（重复片段、明显模板化胡译）

### 4.2 轻量模式提质（轻量 + 本地）
- **术语系统**：Glossary（术语表）+ TM（翻译记忆）+ 一致性校验。
- **结构化断句**：将 ASR 碎片合并成“句子单元”再翻译，最后映射回时间轴。
- **ASR 文本净化**：错别字/专名/数字规范化、基础标点恢复。
- **英文后处理**：规则清洗 +（可选）LanguageTool 语法修正（不改写，只纠错）。
- **分档策略**：lite / lite+（仍本地、仍非 LLM），仅在用户明确选择时启用更强 MT。

### 4.3 质量模式提质（本地、接近线上观感）
- **多步翻译**：忠实（faithful）→ 反思（reflect）→ 改写（adapt），每步都要格式约束与校验。
- **字幕产品规范**：单行优先、长句智能拆分、阅读速度控制。
- **字幕稿 vs TTS 稿分离**：字幕保留信息密度；TTS 版本去脚注/括号/奇怪标记，保证可朗读。
- **一致性强约束**：术语表注入 prompt + 输出后校验（不一致则纠正/重试）。

## 5. 工作流（用户视角）

### 5.1 单任务流程
1. 上传视频（或选择已有素材）
2. 选择模式：轻量 / 质量
3. 选择预设（轻量 normal/mid/high，质量 quality）
4. 点击开始
5. 查看进度（7 步）：提取音频 → ASR → 翻译 → TTS → 合成 → 字幕 → 完成
6. 下载产物：
   - 成片：`output_en.mp4`、`output_en_sub.mp4`
   - 字幕：`chs.srt`、`eng.srt`、`bilingual.srt`
   - 中间产物：`audio.json`、`asr_whispercpp.json`（轻量）等

### 5.2 审校与回归（商业化必须）
- 审校人员可直接对 `eng.srt` 做修改并重新“只做合成/只做字幕封装”。
- 每次发版/改模型/改 prompt 前后，跑 Golden Set 得到对比报告（分数+差异）。

---

## 6. 配置与部署

### 6.1 离线优先
- 默认可在断网环境执行（本地模型齐全时）。
- 在线能力（可选）不得影响离线主链路的稳定性。

### 6.2 可运维性
- 日志规范：每任务独立 `run.log`，包含关键参数快照与版本信息。
- 断点续跑：每步产物存在则跳过，支持从 ASR/翻译/TTS 继续。

---

## 7. 风险与对策（商业化视角）
- **模型文件不齐**：提供模型清单与校验工具（启动前检查缺失）。
- **硬件差异大**：明确给出硬件档位与模式映射，前端按档位给推荐。
- **字幕/配音不一致**：用“时间轴补静音 + 估时裁剪 + TTS 稿分离”降低风险。
- **质量漂移**：Golden Set + 质量门禁 + 版本对比报告。

---

## 8. 指标（衡量提质是否有效）
- **字幕正确性**：Golden Set 的人工评分/自动对齐率
- **一致性**：术语漂移率、同实体多译率
- **可读性**：超长行比例、阅读速度超限比例
- **可交付性**：任务成功率、可恢复成功率、死链/缺产物比例
