## 硬件配置分级（面向 C 端台式机，离线中译英配音）

本项目有两条主要流水线（先选“模式”，再看硬件）：
- **轻量模式（lite）**：whisper.cpp ASR + Marian MT + Piper TTS（最稳、资源需求最低、默认离线）
- **质量模式（quality）**：WhisperX ASR/对齐 + 本地 LLM 翻译（可启 TRA）+ Coqui/Piper TTS（质量上限更高，对 CPU/RAM/GPU 更敏感）

每档统一描述维度：**CPU / 内存 / GPU / 磁盘、适用模式、推荐预设/配置、预估耗时（1 分钟视频）**。

---

## 0. 快速选型（推荐）
- **机器没显有独 / 只有 8–16GB 内存**：优先 **lite**（质量模式也能跑，但更多是“可用”而非“顺滑”）
- **有 NVIDIA GPU ≥8GB（Linux 原生环境）**：可长期使用 **quality**（WhisperX + LLM 更稳、更接近线上观感）
- **macOS / Docker Desktop**：通常无法直接利用宿主 GPU（依赖部署方式），建议把 quality 当作 **CPU 质量模式**来评估（可跑但更慢）

---

## 1. Lite 硬件档位（离线、稳定优先）

### 1.1 Lite 基础档（入门/体验）
- **配置**：CPU 4–8 线程 / 8–16GB RAM / 无独显
- **适用模式**：lite
- **推荐预设**：`presets.normal`
- **推荐模型/参数**
  - ASR：whisper.cpp `ggml-small-q5_0`（更快）或 `ggml-medium-q5_0`（更准）
  - MT：本地 Marian（`assets/models/mt/opus-mt-zh-en`）
  - TTS：Piper（默认完全离线）
  - 可选：开启 VAD（若切得太碎或噪声大，可调 `vad_threshold/vad_min_dur`）
- **预估耗时**：约 5–15 分钟/1 分钟视频（强依赖 CPU 与噪声）
- **适用**：短视频、体验、小批量

### 1.2 Lite 中端档（常规创作）
- **配置**：CPU 8–12 线程 / 16–32GB RAM / 任意 GPU（对 lite 影响不大）
- **适用模式**：lite
- **推荐预设**：`presets.mid`
- **推荐模型/参数**
  - ASR：`ggml-medium-q5_0`（更稳）
  - VAD：建议开启（减少噪声段/空段）
  - 句子单元：可选开启（提升翻译上下文稳定性）
- **预估耗时**：约 2–6 分钟/1 分钟视频
- **适用**：日常使用、较稳定的交付节奏

### 1.3 Lite 高端档（批量/更快）
- **配置**：CPU ≥12 线程 / 32GB+ RAM
- **适用模式**：lite（仍然更稳）
- **推荐预设**：`presets.high`
- **预估耗时**：约 1–3 分钟/1 分钟视频（视 CPU/磁盘）
- **适用**：长视频/批量处理

---

## 2. Quality 硬件档位（质量上限更高，资源敏感）

### 2.1 Quality CPU 档（可跑但偏慢）
- **配置**：CPU ≥12 线程 / 32GB RAM（越大越稳）/ 无 GPU 或 GPU 不可用
- **适用模式**：quality（CPU 跑 WhisperX + LLM）
- **推荐预设**：`presets.quality`（必要时下调模型/参数）
- **推荐参数**
  - WhisperX：`whisperx_model=medium`
  - LLM：`llm_chunk_size` 建议更小（例如 1–2）换稳定
  - P0/P1：建议开启（字幕整形 + TTS 稿分离 + TTS 超时裁剪），提升可交付性
- **预估耗时**：约 4–12 分钟/1 分钟视频（极依赖 CPU、模型、背景噪声）
- **适用**：追求更好的翻译/审校/门禁能力，但接受更慢

### 2.2 Quality GPU 档（推荐）
- **配置**：CPU ≥12 线程 / 32GB+ RAM / NVIDIA GPU ≥8GB（建议 12–16GB）
- **适用模式**：quality（WhisperX/对齐 + LLM）
- **推荐预设**：`presets.quality`
- **推荐参数**
  - WhisperX：可尝试更大模型（在显存允许范围内）
  - LLM：稳定优先时 `llm_chunk_size` 小；追求吞吐再逐步增大
  - 可选：开启 TRA 多步翻译（需要配套评估/回退策略）
- **预估耗时**：约 1–4 分钟/1 分钟视频（取决于 GPU/LLM）
- **适用**：质量模式常规生产、长视频/批量更合适

## 3. “症状 → 怎么调/该升级什么”（最实用）
- **症状：分段变成 2/6 段，字幕很长**
  - **原因**：WhisperX VAD 没开或不生效（vad=off）
  - **处理**：在 quality 打开 `vad_enable`，调 `vad_threshold/vad_min_dur`
- **症状：LLM 经常 500/runner crash 或翻译偶发空行**
  - **原因**：LLM 资源不够（OOM/CPU 压力）或单次请求太大
  - **处理**：减小 `llm_chunk_size`；必要时换更小模型/加内存
- **症状：CPS（阅读速度）大量超限**
  - **处理**：优先开启 P0 的 `subtitle_cps_fix_enable`（能延长时间轴就延长）；仍超限再考虑更强分句/或 P1 的 TTS 裁剪策略（朗读稿维度）
- **症状：配音经常听起来很赶/像被截断**
  - **处理**：开启 P1 `tts_fit_enable`（TTS 稿超时裁剪），并适当降低 `tts_speed_max` 保音质
- **症状：质量模式整体很慢**
  - **优先升级**：NVIDIA GPU（WhisperX/LLM 受益最大）→ 内存（减少抖动/OOM）→ CPU 核数

---

## 4. 说明（口径统一）
- 本项目 lite 默认 **Piper**（完全离线、依赖最少）；quality 默认 **Coqui**（可换 Piper）。
- 耗时受：视频噪声、人声占比、VAD、模型大小、LLM chunk、后台负载等影响，上述为粗估。
- “质量”最终以 `quality_report.json` 门禁指标与 Golden Set 回归为准（避免只看主观观感）。
