## 0. 目标定义（我们要实现什么）

### 0.1 业务目标（给非技术同学）

在不改变“中文 ASR → 翻译 →（可选提质）→ 字幕后处理 →（可选配音）→ 封装交付”的整体体验前提下：

- **支持 3 种目标语（Target Language / 目标语言）**：
  - `en`：英语（English / 英文）
  - `ja`：日语（Japanese / 日文）
  - `ru`：俄语（Russian / 俄文）
- **输入语固定为中文（Chinese / 中文）**：现阶段仍以中文 ASR 为主，不在本次范围内扩展日语/俄语 ASR。

### 0.2 “完成”的可验收定义（Definition of Done / 验收口径）

对任意一段视频任务（Lite/Quality/Online 任一路径）：

- 用户可以在 UI 或 CLI 选择目标语言（en/ja/ru）
- 系统能生成对应目标语言的字幕文件、视频产物（命名规范见第 1 节）
- 质量报告（`quality_report.json`）的门禁规则不会因为目标语不是英语而误判（例如日语不应触发 “english_purity”）
- 审校工作流（review）能对所选目标语字幕进行导入/编辑/回写
- 评测脚本（golden suite）能按目标语读取对应金标并给出 `p_improve` 等置信度指标

## 1. 统一的“多语种产物命名规范”（必须改，避免全项目到处 if）

> 为什么要定命名：当前项目在 scripts/backend/frontend 里大量写死 `eng.srt / eng_tts.srt / output_en.mp4`。要支持日语/俄语，必须**统一“语言无关”的命名方式**，并提供英文老命名的兼容（避免历史功能全断）。

### 1.1 新的标准产物命名（最终对外 / 对内都统一）

以 `tgt_lang` 表示目标语（`en|ja|ru`）：

- **主目标字幕（交付字幕）**：`sub.<tgt_lang>.srt`  
  例：`sub.en.srt` / `sub.ja.srt` / `sub.ru.srt`
- **目标字幕审校版（review）**：`sub.<tgt_lang>.review.srt`
- **展示字幕（display）**：`sub.<tgt_lang>.display.srt`
- **配音朗读稿（TTS script）**：`tts.<tgt_lang>.srt`  
  - 原始朗读稿（若保存）：`tts.<tgt_lang>.raw.srt`
- **双语字幕（bilingual）**：`bilingual.<tgt_lang>.srt`（中文 + 目标语）
- **视频交付产物**：
  - 配音视频：`output_<tgt_lang>.mp4`
  - 配字幕视频：`output_<tgt_lang>_sub.mp4`

### 1.2 兼容旧命名（必须保留，保证现有流程不崩）

当 `tgt_lang=en` 时，额外写一份“旧文件名”：

- `eng.srt` == `sub.en.srt`
- `eng.review.srt` == `sub.en.review.srt`
- `eng_display.srt` == `sub.en.display.srt`
- `eng_tts.srt` == `tts.en.srt`
- `eng_tts_raw.srt` == `tts.en.raw.srt`
- `bilingual.srt` == `bilingual.en.srt`
- `output_en.mp4` == `output_en.mp4`（保持不变，但内部一律按 `output_<tgt_lang>.mp4` 读写）
- `output_en_sub.mp4` == `output_en_sub.mp4`

> 中文说明：这样做的好处是——**英文用户完全无感**；而新增日语/俄语不需要在系统里“假装叫 eng.srt”。
