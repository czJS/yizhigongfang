# 前端页面改版方案（最优交付体验版 / 串行队列 / 批次看板 / 交付向导 / 中文友好 / DevOnly / 自定义输出目录）

本文档是“译制工坊”前端的**彻底改版设计**（不包含代码实现）。目标是把当前“单任务、超长单页、工程参数堆叠”的界面，升级为一个真正面向交付的产品：**用户只需要关心导入、输出目录、交付物与失败处理**，而不需要理解模型、路径、日志细节。

本方案采用 **AB 混合**的最优形态：
- **A：交付向导（Wizard）**：创建批次时把关键问题一次解决（导入、输出目录、交付配方、预设）。
- **B：批次看板工作台（Kanban）**：执行阶段以状态与行动为中心（待处理/处理中/已完成/失败），让批量交付像流水线一样顺。

---

## 0. 设计前提（硬约束）

- **串行队列**：一次只处理一个任务（最稳）。
- **失败不阻塞**：某个任务失败后继续下一个任务。
- **默认交付配方**：默认交付“成片（带字幕）+ 字幕（中/英）+ 质量摘要”。
- **输出目录必选**：交付物默认不落到 `outputs/` 这种内部目录，对用户不可见。
- **中文友好**：短句、动词开头、少术语、给下一步动作。
- **DevOnly**：调试/诊断/内部信息仅在开发模式显示。

---

## 1. 目标（以“高效完成交付”为唯一北极星）

### 1.1 用户可感知目标

- **10 秒上手**：新用户一眼找到“导入 → 选择输出 → 开始 → 拿成片”。
- **批量更快**：多视频导入后，不需要频繁切页面/找按钮，一次把批次跑完。
- **交付直达**：任何时候都能清晰看到“交付物在哪里、如何打开/下载”。
- **失败可行动**：失败只给“一句话原因 + 下一步按钮”，不逼用户读日志。
- **界面不吵**：默认只显示与交付相关的信息；高级/诊断按需展开。

### 1.2 非目标（暂不做）

- 不做并行执行与并行度控制（明确不支持）。
- 不做复杂账户/权限体系。
- 不把评测套件（E2E/金标）强行塞进交付主流程（可另做“实验/评测”入口）。

## 2. 核心信息架构（IA）

### 2.1 顶层结构

- **新建批次（Wizard）**
- **批次工作台（Kanban + 详情抽屉）**
- **设置**
  - 普通设置（对用户有意义）
  - 开发者工具（DevOnly）

### 2.2 “默认视图只给交付”分层规则

- **默认可见（所有用户）**
  - 导入/队列/进度/交付物/失败原因（人话）
  - 输出目录（可打开）
  - 少量“交付相关”的开关（如双语字幕）
- **高级设置（默认折叠）**
  - 预设档位、质量/速度倾向、少量关键参数
- **开发者工具（DevOnly）**
  - 模型名、内部路径、全量日志、全量中间产物、原始 JSON、接口调试

## 3. 页面 1：新建批次（Wizard）——创建即可交付的生产单

### 3.1 目标

用最少步骤得到一个“可运行、可交付、可追溯”的批次：导入清单 + 输出目录 + 交付配方 + 预设。

### 3.2 向导步骤（推荐 3 步）

#### Step 1：选择视频（导入清单）

- 支持：
  - `选择多个视频…`
  - `选择文件夹…`（自动扫描常见格式）
  - `拖拽导入`
- 列表显示（每条任务）：
  - 序号（=处理顺序）
  - 文件名（主标题）
  - 时长/分辨率（可异步探测；失败不阻塞创建）
- 列表操作：
  - 删除/清空
  - 拖拽排序（直接决定队列顺序）

#### Step 2：批次设置（一次性定好交付关键项）

- **批次名**：默认 `批次-YYYYMMDD-HHMM`（可编辑）
- **输出位置（必填）**：`选择文件夹…`
  - 提示：`成片和字幕会保存到这里`
  - 校验：不可写则阻止继续，并提示 `无法写入该文件夹`
- **交付配方（默认）**：交付成片（Delivery）
  - 简短说明：`自动生成成片（带字幕）+ 字幕文件`
- **运行预设（默认推荐）**：用“人话”呈现，而不是硬件/模型术语：
  - `更快（推荐）`：更省时间
  - `更清晰`：更高质量（更慢）
  - `更省资源`：适合低配
  - 备注：可在“高级设置”查看细节

#### Step 3：确认并开始

- 展示一个“交付确认卡片”：
  - 视频数量、输出目录、交付配方、预设
- 操作：
  - `开始处理`（主按钮）
  - `稍后开始`（创建批次但不启动）

## 4. 页面 2：批次工作台（Kanban）——批量交付流水线

### 4.1 设计目标

让用户在一个页面里完成：
- 看清队列状态（现在处理谁、还剩多少）
- 快速处理失败（查看原因/续跑/重试/跳过）
- 快速交付完成结果（打开成片/打开输出目录/批次汇总）

### 4.2 结构总览（文字线框）

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 顶栏（固定）                                                                 │
│ 批次：批次-20251231-1030 [编辑]  输出：D:\交付\某客户\ [打开] [更改…]        │
│ 总进度：3/9  完成：2  失败：1  待处理：6                                    │
│ 当前：003_yyy.mp4  状态：处理中  步骤：翻译中…  [进度条]                    │
│ [开始/暂停/继续]  [跳过当前]  [重试当前]  [打开输出目录]  [新建批次]        │
└──────────────────────────────────────────────────────────────────────────────┘
┌──────────────────────────────────────────────────────────────────────────────┐
│ 看板（Kanban，横向 4 列）                                                    │
│  待处理（6）     处理中（1，最多1个）     已完成（2）        失败（1）        │
│  [任务卡片...]    [当前任务卡片]          [任务卡片...]      [失败卡片...]    │
│                                                                              │
│  点击任意卡片 → 右侧详情抽屉（交付物优先）                                   │
└──────────────────────────────────────────────────────────────────────────────┘
```

### 4.3 看板列定义（用户能理解的状态）

- **待处理**：未开始的任务（按顺序）
- **处理中**：当前正在处理（最多 1 个）
- **已完成**：可交付（默认意味着通过质量门禁；若允许“完成但不建议交付”，见下）
- **失败**：任务失败但队列继续（每条失败必须有“一句话原因”）

> 可选：把“已完成但不建议交付”作为已完成列里的黄色标签（不新增列，减少复杂度）。

### 4.4 任务卡片（最小信息 + 直接行动）

每张卡片只显示：
- 文件名（主标题）
- 状态标签（待处理/处理中/已完成/失败）
- 一句话摘要：
  - 处理中：`正在翻译…` / `正在合成配音…`
  - 已完成：`可交付`
  - 失败：`失败：字幕格式不符合要求（点查看原因）`
- 主要按钮（因状态变化）：
  - 待处理：`置顶（改顺序）`（可选）
  - 处理中：`暂停`（或仅顶栏控制）
  - 已完成：`打开成片` `打开输出`（两按钮够用）
  - 失败：`查看原因` `从上次继续`（或 `重试`）

## 5. 任务详情抽屉（点击卡片出现，交付物永远第一）

### 5.1 抽屉 Tabs（固定顺序）

1) **交付物**（默认打开）  
2) **概览**（任务记录与时间）  
3) **质量检查**（问题清单，可定位）  
4) **审校（可选）**（字幕查看/编辑/导入导出）  
5) **日志**（默认折叠，DevOnly 可更强）  
6) **更多文件（诊断）**（默认隐藏，DevOnly 或“导出诊断包”才出现）

### 5.2 交付物 Tab（最重要）

只展示对交付有意义的 3~4 个文件，并给一键操作：
- **成片（带字幕）**：`打开` / `复制路径`
- **字幕（中文）**：`打开` / `下载`
- **字幕（英文）**：`打开` / `下载`
- **质量摘要**：`查看`（界面内摘要 + 通过/不通过）

若任务失败：
- 顶部红色提示：`当前任务未完成，暂无可交付文件`
- 给下一步按钮：`查看原因` / `从上次继续`

### 5.3 质量检查 Tab（把“失败”变成可修复）

以清单形式展示（中文短句）：
- 通过/不通过
- 主要问题 Top3（最多 3 条）
- 详细问题列表：每条问题可点击定位到具体字幕条目（若支持）
- 一键动作：
  - `从上次继续（推荐）`
  - `重试（从头）`

### 5.4 审校 Tab（可选但建议规划）

目标：把“交付前最后一步”放在产品里，而不是让用户去外部找文件。
- 字幕查看（中/英切换）
- 导入外部审校字幕（覆盖/对比）
- 保存审校版本
- 一键应用：`应用审校并重新生成成片`

## 6. 队列控制与异常策略（串行但不脆弱）

### 6.1 顶栏队列控制（始终可见）

- `开始`：从队首或上次位置开始
- `暂停`：暂停当前任务（若支持）
- `继续`：从暂停点继续
- `跳过当前`：把当前标记为失败/跳过，进入下一个（需要二次确认）
- `重试当前`：对当前任务执行重试（默认优先“从上次继续”）
- `打开输出目录`：打开 `<输出目录>/<批次名>/`

### 6.2 失败的产品化表达（必须做到）

失败卡片必须满足：
- **一句话原因**：用户读得懂（不要堆栈/路径/模型）
- **一个明确下一步**：查看原因 / 从上次继续
- **不阻塞**：失败不影响队列继续

### 6.3 续跑（Resume）策略（对用户只暴露“从上次继续”）

对用户只给一个按钮：`从上次继续（推荐）`  
内部可映射为从某阶段恢复（asr/mt/tts/mux 等），但这些属于 DevOnly 细节。

## 7. 输出到用户指定目录（交付物第一原则）

### 7.1 交互规则

- 创建批次时必须选择输出目录（不可跳过）。
- 任何“内部目录/容器路径/outputs/uploads”等内容不应在用户默认视图出现。
- UI 默认只说：`保存到：<用户选择的目录>`

### 7.2 推荐目录结构（稳定、可批量交付、可追溯）

```text
<输出目录>/<批次名>/
  001_<视频名stem>/
    成片_带字幕.mp4
    字幕_中文.srt
    字幕_英文.srt
    质量摘要.json
    诊断包.zip            （按需生成）
  002_<视频名stem>/
  ...
  批次汇总.md             （批次结束自动生成，用户友好）
```

说明：
- `001_` 是队列顺序（=处理顺序）。
- `<视频名stem>` 建议做安全化（去除特殊字符）。
- “诊断包”仅用户显式选择时生成（或 DevOnly 自动提供）。

## 8. 交付配方（Delivery）与可选项

### 8.1 默认交付物（所有用户可见）

- 成片（带字幕）
- 字幕（中文）
- 字幕（英文）
- 质量摘要（界面可读）

### 8.2 可选交付项（高级设置）

- 同时导出双语字幕
- 导出审校字幕（review 版本）
- 导出诊断包（日志+报告+关键中间文件）

---

## 9. 文案与信息呈现规范（中文友好必须落地）

### 9.1 文案原则

- **短句 + 动词开头**：选择、开始、暂停、继续、打开、重试、跳过、导出
- **避免术语**：默认不用 QE/chrF/VAD/embedding/模型名/容器路径
- **给下一步**：任何状态都要让用户知道下一步点哪里

### 9.2 推荐文案（可直接复用）

- 向导 Step1 提示：`先把视频加进列表（顺序就是处理顺序）`
- 输出目录提示：`成片和字幕会保存到这里`
- 运行中提示：`处理中…你可以先去做别的，完成后这里会提示。`
- 失败提示：`失败：字幕检查未通过（点“查看原因”）`
- 完成提示：`完成：点击「打开成片」交付，或打开输出目录。`

## 10. DevOnly（开发者工具）规则

### 10.1 DevOnly 默认隐藏内容

- 模型名、内部路径、endpoint、完整配置 JSON
- 全量日志（可保留“最近 20 行摘要”给普通用户）
- 中间产物浏览器（大量 wav/json）
- 调参项（线程、阈值、设备选择等细粒度参数）

### 10.2 DevOnly 入口

- 仅开发环境显示：`开发者工具`
- 默认折叠，文案：`仅用于排查问题，不影响交付`

---

## 11. 验收清单（上线前必须全过）

- 新用户是否能在 10 秒内完成：导入 → 选输出目录 → 开始？
- 批量任务是否能在一个页面里顺畅跑完（不需要到处找按钮）？
- “交付物”是否永远在最显眼位置，且一键打开输出目录？
- 失败是否不阻塞队列？失败卡片是否有“一句话原因 + 下一步按钮”？
- 默认界面是否不出现内部路径、模型名、复杂参数？
- 输出目录结构是否稳定、不覆盖、可追溯？
- DevOnly 是否在生产环境完全不可见？

## 12. 附录：概念模型（用于前后端与 UI 对齐）

### 12.1 批次（Batch）

- id
- name（可编辑）
- output_dir（必填）
- created_at / started_at / ended_at
- status：未开始 / 进行中 / 已完成（全部完成或失败都结束）
- counters：total / done / failed / pending

### 12.2 任务（Task）

- id
- index（001、002…）
- input_video（原始文件）
- status：待处理 / 处理中 / 已完成 / 失败 / 已暂停
- stage（内部阶段，DevOnly）
- progress（0-100）
- message（人话摘要）
- failure_reason（一句话原因，必须可读）
- resume_hint（是否可从上次继续）
- deliverables（交付物清单）
