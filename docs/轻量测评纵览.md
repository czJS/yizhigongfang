## 2) Round1（单开关）快筛结论（short3）

> short3 的定位：只用于快速排雷/发现明显收益，不做最终默认策略结论。

本次 short3 结果（所有 run 均 passed_rate=1.0）：

- **baseline**：e2e_mean=97.8
- **asr_normalize_off**：e2e_mean=97.8（与 baseline 无差异）
- **bilingual_srt_off**：e2e_mean=97.8（与 baseline 无差异）
- **denoise_on**：e2e_mean=96.6（略降）
- **vad_on**：e2e_mean=97.8（无差异）
- **sentence_unit_on**：e2e_mean=97.4（略降）
- **entity_protect_on**：e2e_mean=97.8（无差异）

解释（为什么 short3 看起来“差不多”）：
- short3 很短、样本只有 3 段，很多能力（例如 VAD/专名保护）在这种样本上触发不足，差异不一定显著。
- 因此这里的结论只能用于：**没有明显的“通过率红线”问题**；是否默认开，需要 golden9 复核。

## 3) Round1.5（三点扫参）快筛结论（short3）

本次对以下参数做了三点扫参（P/D/A），结果在 short3 上 **e2e_mean 全部=97.8**（未观察到差异）：
- `vad_threshold`：0.5 / 0.6 / 0.7
- `vad_min_dur`：0.8 / 1.5 / 2.0
- `min_sub_duration`：1.0 / 1.5 / 2.0
- `tts_split_len`：60 / 80 / 120
- `tts_speed_max`：1.05 / 1.10 / 1.20

解释：
- 这类参数往往需要更长样本、更多样本类型才会体现差异（尤其是 TTS 对齐压力、CPS 告警、风险命中等）。
- 建议下一步把扫参放到 golden9（或至少扩到更大的 short 集）。
