# 质量模式文档总览

> 这套目录是**质量模式最终保留文档集合**。  
> 目标：只保留当前产品、交付、测试、配置、架构、规则、实验真正需要长期维护的版本。  
> 口径：**第一阶段只考虑 Windows 打包交付**；质量模式依赖**本地模型 + 本地 Ollama + 单机运行**。

---

## 文档结构

- `产品概览.md`
  - 质量模式是什么、当前支持什么、不支持什么、用户会得到什么。
- `重构优化最佳实践.md`
  - 总结本轮质量模式重构、优化、专项测试的收口经验，供后续轻量模式复用。
- `交付与Windows打包.md`
  - 质量模式如何打成 Windows 包、打包机需要准备什么、交付件是什么。
- `质量门禁与测试.md`
  - 发版门禁、`quality_report.json` 语义、人工与自动化回归怎么做。
- `配置说明.md`
  - `configs/defaults.yaml` 与 `configs/quality.yaml` 的合并规则与关键字段。
- `运行架构.md`
  - 质量模式从 Electron 到 backend 到 pipeline 到 worker 的运行闭环与目录职责。
- `规则中心.md`
  - 规则中心的定位、作用范围、用户怎么确认规则生效。
- `实验评测.md`
  - 研发实验与版本比较资料，明确与发版门禁的边界。

---

## 阅读顺序

如果你只想快速理解质量模式，建议按下面顺序读：

1. `产品概览.md`
2. `重构优化最佳实践.md`
3. `交付与Windows打包.md`
4. `质量门禁与测试.md`
5. `配置说明.md`
6. `运行架构.md`

如果你要做研发实验，再继续看：

7. `实验评测.md`

如果你要维护术语与规则：

8. `规则中心.md`

---

## 不再保留的文档类型

这次整理后，以下类型不再作为质量模式长期文档保留：

- 旧路径跳转页
- 历史 Round1 / sweep 手册
- 自动生成测试报告
- 一次性重构任务单
- 已被正式文档吸收的阶段性方案稿

---

## 当前总原则

- 产品文档只讲**当前代码真实支持**的质量模式。
- 发版文档只讲**这版能不能交付**。
- 实验文档只讲**研发如何比较版本与参数**。
- 不再混用“产品口径”和“实验口径”。
