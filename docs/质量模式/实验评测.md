# 质量模式实验评测

> 本文只服务于**研发实验、版本比较、参数决策**。  
> 它不承担当前产品发版门禁职责。  
> 如果你要判断“这版能不能交付”，请看 `质量门禁与测试.md`。

---

## 1. 实验评测回答什么问题

研发实验主要回答三件事：

1. 某个策略是否真的带来提升
2. 提升发生在什么层面
3. 提升是否稳定可信

## 2. 当前实验评测分层

### 2.1 门禁底线

来源：

- `quality_report.json`

作用：

- 判断能否交付
- 判断是否存在明显风险

### 2.2 E2E 体验分

来源：

- `pipelines/lib/eval/eval_quality_e2e_suite.py`

作用：

- 评估端到端交付体验
- 比较不同开关、参数、版本

### 2.3 金标绝对分

来源：

- `pipelines/tools/eval_quality_e2e_golden_suite_impl.py`

作用：

- 比较 ASR 识别质量
- 比较翻译文本质量
- 给“为什么变好/变差”提供更可解释证据

## 3. 当前实验数据集

### 3.1 `golden9`

定位：

- 主评测集
- 适合 ASR / MT / TTS 主链路实验
- 支持金标绝对质量对比

### 3.2 `short3`

定位：

- 快速体验集
- 适合展示字幕、擦字幕、封装类实验

### 3.3 `fluency_en`

定位：

- 文本回归集
- 适合 prompt、术语、分句等主要影响文本的改动
- 不覆盖 TTS、mux、embed 的多媒体体验

## 5. 典型实验流程

### 5.1 做 E2E 对比

1. 准备 `segments.jsonl`
2. 准备 `experiments.yaml`
3. 运行 `run_quality_e2e_impl.py`
4. 运行 `eval_quality_e2e_suite.py`
5. 对比 `passed_rate`、`e2e_score_100`、成本与失败率

### 5.2 做文本自然度对比

1. 构建 `cases.jsonl`
2. 跑 baseline 与候选版本
3. 用 `eval_fluency_suite.py` 出报告
4. 对比 `final_score_100` 与 `p_improve`

### 5.3 做 ASR CER 对比

1. 准备 `ref_zh` / `pred_zh`
2. 运行 CER 评测脚本
3. 对比 CER 与 `asr_score_100`

---

## 6. 当前推荐读结果顺序

1. 先看 `passed_rate`
2. 再看 `e2e_score_100`
3. 有金标时再看 ASR / MT 绝对分
4. 最后看 `p_improve`

原因：

- 通过率掉了通常说明方案不稳定
- E2E 分最接近交付体验
- ASR / MT 分更适合解释原因
- `p_improve` 用来判断结论是否可信

## 7. 实验评测不替代什么

实验评分不能替代：

- Windows 打包自检
- 本地 Ollama / 模型包 / 授权门禁检查
- 人工听感与观感抽检
- 当前发版最小检查

这些统一看：

- `交付与Windows打包.md`
- `质量门禁与测试.md`

#### 4.5.2 可三点扫参候选点

##### ASR

- `vad_*`：已从当前质量模式产品主线删除，仅代表历史 Round1.5 实验，不再作为现行参数口径
- `asr_merge_short_max_gap_s`：短段合并最大间隔（CLI：--asr-merge-short-max-gap-s）→ **P/D/A：0.15 / 0.25 / 0.40**

##### MT

- `llm_chunk_size`：LLM 分块大小（CLI：--llm-chunk-size）→ **P/D/A：1 / 2 / 3**
- `mt_context_window`：翻译上下文窗口（CLI：--mt-context-window）→ **P/D/A：0 / 1 / 2**
- `mt_max_words_per_line`：单行英文最大词数（CLI：--mt-max-words-per-line）→ **P/D/A：14 / 18 / 22**

##### Display

- `display_max_chars_per_line`：显示字幕单行最大字符数（CLI：--display-max-chars-per-line）→ **P/D/A：36 / 42 / 48**
- `display_merge_max_gap_s`：显示字幕合并最大间隔（CLI：--display-merge-max-gap-s）→ **P/D/A：0.15 / 0.25 / 0.35**

##### TTS / Mux

- `tts_speed_max`：最大提速倍率（CLI：--tts-speed-max）→ **P/D/A：1.04 / 1.08 / 1.12**
- `mux_slow_threshold_s`：触发慢放阈值（CLI：--mux-slow-threshold-s）→ **P/D/A：0.6 / 1.0 / 1.4**
