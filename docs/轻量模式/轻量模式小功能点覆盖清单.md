# 轻量模式小功能点覆盖清单

> 本文按当前产品口径梳理轻量模式所有用户可见的小功能点，并映射到现有自动化测试与人工测试。  
> 目标不是替代 `自动化测试用例.md` 或 `人工测试用例.md`，而是作为更细粒度的覆盖底账，帮助后续查漏补缺。

---

## 1. 统计口径

- 只统计当前轻量模式下用户可见、可操作、可感知的功能点
- 不统计隐藏入口、历史遗留 E2E 资产、纯内部实现细节
- 后台能力只有在当前 UI 有明确入口或直接支撑用户可见行为时，才计入本表

### 1.1 状态定义

| 状态 | 含义 |
| --- | --- |
| `已有自动化` | 已有较明确的自动化断言覆盖 |
| `部分自动化` | 已有页面级或契约级覆盖，但未覆盖到该细项本身 |
| `仅人工` | 当前只能由人工测试或人工抽检兜底 |
| `未纳入` | 当前未见自动化，也未在人工用例中单列 |

### 1.2 相关入口

- 前端页面：`apps/desktop/src/screens/`
- 任务详情抽屉：`apps/desktop/src/components/taskDrawer/TaskDrawerContent.tsx`
- 清理弹窗：`apps/desktop/src/components/modals/CleanupArtifactsModal.tsx`
- 轻量参数过滤：`apps/desktop/src/app/domains/params/batchParams.ts`
- 轻量任务创建：`apps/desktop/src/hooks/useWizardBatchCreation.ts`
- 后端 API：`apps/backend/backend/app.py`

## 3. 当前缺口结论

### 3.1 已经比较稳的区域

- 模式选择基础流转
- `Wizard` 主要创建与确认链路
- 历史记录
- 规则中心真实编辑器交互
- 高级设置的 lite 核心参数真实编辑与默认值保存
- 系统页 lite 基本文案
- 轻量核心任务 API 契约

### 3.2 仅覆盖到页面级、但没测透细项的区域

- `Wizard`：硬擦除滑条实际写回细节仍偏回调级，未做更完整区域联动断言
- `Workbench`：抽屉内部真实内容仍未测透
- `TaskDrawerContent`：`交付检查` 明细展开、更多自助修复分支仍缺组件测试

### 3.3 当前最大缺口

- `TaskDrawerContent`：质量报告结论展示、一键从识别开始重试、一键补封装硬字幕仍偏人工或间接覆盖
- `Workbench`：抽屉内真实内容与批次更多组合态
- `Wizard`：硬擦除区域联动与更完整回写细节
- Windows 打包态链路：首次安装、首次启动、覆盖安装、卸载、打包态诊断包与输出目录真实行为

---

## 4. 建议补测顺序

### 4.1 自动化优先

1. `TaskDrawerContent`：质量报告结论、自助修复剩余分支
2. `WizardScreen`：硬擦除区域联动与滑条写回细节
3. `WorkbenchScreen`：抽屉内真实内容与更多批次组合态
4. Windows 打包态链路：安装、输出目录、诊断包、升级、卸载

### 4.2 人工补位

以下不建议只靠自动化判定，仍应保留人工抽检：

- 硬擦除区域是否真的遮挡合理
- 自助修复后产物是否真正可交付
- 失败提示是否对普通用户易懂
- 审核页改写后的字幕是否更易读
