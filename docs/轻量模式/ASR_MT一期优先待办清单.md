# ASR / MT 一期优先待办清单

> 目标：只保留一期**收益大、时间成本合理、能直接改善主链结果**的事项。  
> 这不是“所有可能优化项”的大全，而是一份可排期、可执行、可验收的短清单。

---

## 0. 先给结论

一期最值得继续做的，不是再大范围试模型，而是：

1. 把 `ASR 项目混淆正式集` 持续扩到更好用的规模，并固定 before/after 对比。
2. 把 `来源视频筛样质量` 做好，提高每 500 个视频能产出的高价值条目数。
3. 把 `MT 已实现但未定案` 的两项低成本能力做成有证据的决策：
   - `sentence_unit`
   - `entity_protect`

不建议一期继续投入的方向：

- 重开一轮 ASR 主模型大战
- 把 Lite 默认链路接上重型 ASR LLM 修复
- 做全量 MT 后置小 LLM 润色
- 无边界试 generation 参数

## 2. 一期建议做的 ASR 待办

### A1. 继续扩充正式集到 `200-300` 条核心正式项

优先级：`P0`

为什么现在做：

- 这是当前最直接改善识别链路的低成本手段
- 不需要换模型
- 已经有发布脚本、归档文档、对比脚本，基础设施已齐

执行内容：

1. 新增一批高信息密度口播来源视频
2. 继续跑混淆挖掘
3. 人工终审收口
4. 用 `publish-reviewed-final` 发布
5. 用 `compare_project_confusion_assets.py` 做前后对比

验收标准：

- 正式集从当前 `116` 个唯一项扩到下一阶段目标
- 不回流常用词误触发
- before/after 对比报告可读

产物：

- 新版 `assets/zh_phrase/asr_project_confusions.json`
- 对比报告
- 归档文档追加记录

### A2. 做来源筛样优化，把“500 视频 -> 高价值条目产出率”提上来

优先级：`P0`

为什么现在做：

- 你当前已经明确感知到人工审核成本高
- 如果来源样本不准，后续所有人工都在为噪声买单

执行内容：

1. 继续收紧 `speech_focused` / 来源 profile
2. 对比不同视频类型的有效产出率
3. 沉淀一套优先采样类别清单

验收标准：

- 同样数量的视频，高价值人工候选比例更高
- 单轮人工审核噪声明显下降

产物：

- 采样策略说明
- 各类来源产出率对比表

### A3. 把正式集对比固定到代表性语料

优先级：`P1`

为什么现在做：

- 现在已经有固定对比脚本
- 但还缺一套项目内稳定语料，后续每次换正式集都能直接复测

执行内容：

1. 选一份稳定 `transcripts.jsonl` 代表集
2. 固化成对比输入
3. 后续每次正式集更新都跑一遍

验收标准：

- 任意两版正式集都能快速比较
- 不再靠人工临时 spot-check 才知道有没有退化

产物：

- 固定 compare corpus
- 固定 compare 命令模板

### A4. `whisper.cpp` 的线程自动裁剪

优先级：`P1`

为什么值得做：

- 当前还是经验分档值
- 这是产品化问题，不是实验性问题

执行内容：

1. 按 CPU 核心数自动上限裁剪线程
2. 保持 `normal / mid / high` 作为上层预设语义

验收标准：

- 弱机器不再因为线程过高拖慢
- 强机器不再被经验值过度限死

产物：

- 线程策略实现
- 小规模 smoke / regression 报告

## 3. 一期建议做的 MT 待办

> 严格说，MT 真正“完全没做”的高收益项并不多。  
> 一期更值得做的是：把**已实现但未定案**的低成本能力做成明确决策。

### M1. 用代表性集对 `sentence_unit` 做一次决策性复测

优先级：`P0`

当前状态：

- 已实现
- 默认关闭
- short3 上看不出明显收益
- `2026-03-30` 已基于 `cn20` 完成决策性复测
- 当前结论：`继续默认关闭`
- 结果报告：`reports/lite_phase1/mt_feature_retest_cn20/summary.md`

为什么值得做：

- 它直接对应中文源过碎、上下文过短的问题
- 这是当前 MT 侧最现实、最便宜的提升方向之一

执行内容：

1. 在代表性集上跑 `sentence_unit_off / on`
2. 只看是否值得默认开或继续保持高级开关

验收标准：

- 给出明确结论：`默认关 / 高级开关 / 默认开`
- 有数据支撑，不再停留在经验判断

### M2. 用专名密集语料对 `entity_protect` 做一次决策性复测

优先级：`P0`

当前状态：

- 已实现
- 默认关闭
- 价值高度依赖素材类型
- `2026-03-30` 已先在 `cn20` 上完成第一轮决策性复测
- 当前结论：`继续默认关闭，但保留高级开关`
- 当前 `cn20` 结果显示专名候选数为 `0`，说明这批素材不适合作为是否默认开启的最终依据
- 结果报告：`reports/lite_phase1/mt_feature_retest_cn20/summary.md`
- `2026-03-30` 已补充专名密集小集并完成第二轮专项复测
- 专项小集：`reports/lite_phase1/mt_entity_dense_dataset/mt_cases.jsonl`
- 专项报告：`reports/lite_phase1/mt_entity_dense_retest_v2/summary.md`
- 第二轮结论：
  - 当前产品内置的保守抽取在专名密集小集上依旧 `0` 命中
  - 仅用于实验的宽松抽取虽然能抓到 `龙叔 / 典御长` 等候选，但翻译副作用仍明显
  - 因此本期结论继续维持：`默认关闭`，`保留高级开关`，`不进入默认主线`
- `2026-03-30` 已完成一轮“候选抽取策略重构”并回到专名密集小集复测
- 重构要点：
  - 不再只认“引号 / 地名后缀”，而是组合 `引号短语 + 机构/地名后缀 + 姓名/称谓 + 昵称式称谓 + 跨段重复短语`
  - 从“单行命中”改为更看重“跨段重复 + 候选边界干净”，并过滤 `各个 / 这个 / 那个 / 不准他` 这类泛指或动作短语
  - 新增目标侧可用性过滤，挡掉 `Sir.` 这类泛化实体翻译
- 重构复测报告：`reports/lite_phase1/mt_entity_dense_retest_v5/summary.md`
- 第三轮结论：
  - 正式抽取已从旧策略的 `0` 命中提升到 `2` 个 case、`3` 个候选，能稳定抓到 `龙叔 / 中情局 / 典裕掌` 这类更像真实专名的候选
  - 但在强 ASR 噪声样本上仍会带来明显翻译副作用，尤其是 `典裕掌 -> Yuen yeon` 这类错误音译
  - 因此当前最合理的产品结论仍然是：`默认关闭`，`保留高级开关`，`不进入默认主线`

为什么值得做：

- 它直接对应专名漂移、称谓不一致问题
- 比换重模型便宜得多

执行内容：

1. 选择人名 / 地名 / 品牌 / 产品型号密集素材
2. 跑 `entity_protect_off / on`
3. 只判断是否值得作为高级开关保留、是否需要更好的默认阈值

下一步重点：

- 不再用当前 `cn20` 结果尝试把 `entity_protect` 推成默认开
- 候选抽取策略重构已完成，本轮已回到专名密集素材验证
- 如果后续继续做，下一步不再是补普通复测，而是补“ASR 噪声感知 / 低置信专名回避”这一层，再回到这套专名密集小集验证

验收标准：

- 输出“在哪类素材上值得开”
- 输出“是否值得继续保留为产品入口”

### M3. 固化 MT 代表性复测基线

优先级：`P1`

为什么值得做：

- 目前很多 MT 判断还靠阶段性实验记忆
- 一旦后续再调规则，很容易失去统一判断标准

执行内容：

1. 固定 `cn20`、`golden9`、`fluency` 的使用边界
2. 固定 winner/baseline 对比口径

验收标准：

- 后续任何 MT 变更都知道该跑哪一套数据

## 4. 一期明确不做或暂缓的项

### 4.1 ASR 暂缓

1. Lite 主链继续保持无小模型依赖，不再接入轻量小模型修复链路
2. 把 `asr_preprocess` 直接改成 Lite 默认开启
3. 把 `asr_merge_short` 直接改成 Lite 默认开启
4. 重开 ASR 大模型替换路线

原因：

- 这些都已经有实现，或者已经验证过方向
- 但不属于一期最省时、最高收益的动作

### 4.2 MT 暂缓

1. 全量后置小 LLM 润色
2. `mt_context_window` 大范围扫参
3. `glossary_prompt_enable`、`mt_topic_auto_enable` 全面产品化
4. 继续堆 generation 参数
5. 把 `CTranslate2` 直接切进默认主链

原因：

- 收益不确定
- 容易显著增加链路复杂度
- 一期时间成本过高

---

## 5. 推荐排期顺序

如果只按一期收益排序，建议顺序是：

1. `A1` 正式集继续扩到 `200-300`
2. `A2` 来源筛样优化
3. `M1` sentence_unit 决策性复测
4. `M2` entity_protect 决策性复测
5. `A3` 固定 compare corpus
6. `A4` whisper.cpp 线程自动裁剪

---

## 6. 一句话原则

一期不要再追求“什么都试一下”，而要优先做：

> 不换主模型、不过度增重、能直接提升识别质量或人工效率、并且可以稳定验收的事情。
