# 轻量模式文档入口

> 这份目录用于收口轻量模式的正式文档入口。  
> 目标不是“把所有历史材料都搬进来”，而是让产品、研发、测试、交付都能快速找到当前应该看的那几份文档。

---

## 1. 正式入口

建议优先从下面几份文档开始看：

当前一期已经形成的默认结论先放在这里，方便快速对齐：

- `ASR`：保留 `whisper.cpp`，当前默认参数已经落到正式配置与后端透传，不是只针对单机实验
- `离线翻译`：保留 `Marian opus-mt-zh-en`，且 lite 主链已接入批量推理版
- `离线翻译 + 小 LLM`：一期不进默认主链
- `TTS`：已切换为 `Kokoro ONNX` 默认主线，当前默认参数为 `af_bella + en-us + speed=1.0`

- `产品概览.md`
  - 面向产品、运营、交付、测试
  - 用人话说明轻量模式当前卖什么、产出什么、边界在哪里

- `../轻量模式重构方案.md`
  - 面向产品、研发、测试、交付
  - 说明轻量模式为什么要重构、主链如何收口、后续怎么推进

- `ASR_LLM_TTS备选方案详细对比.md`
  - 面向产品、研发、测试
  - 说明轻量模式在 ASR、翻译/LLM、TTS 三个能力层上的主方案与备选方案

- `一期评测脚本设计说明.md`
  - 面向研发、测试、方案负责人
  - 说明一期 `3 选 1` 评测脚本的职责、输出 JSON 结构、当前测试集和汇总决策模板

- `商业化交付方案.md`
  - 面向产品、运营、销售、交付
  - 说明轻量模式在商业体系里的角色、边界和升级逻辑

- `交付与Windows打包.md`
  - 面向研发、交付、打包维护者
  - 说明轻量模式一体化安装包的目标口径、当前仓库阻塞检查、打包机执行步骤和最小验收

- `文档清理清单.md`
  - 面向研发、产品、维护者
  - 记录本轮文档整理中哪些文件被保留、删除、迁移或降级

## 3. 当前文档治理口径

轻量模式文档后续按四类管理：

- 产品文档
- 配置与使用文档
- 测试与验证文档
- 研发/重构文档
- 商业化与交付文档

不再建议继续增加：

- 同主题重复概览
- 只反映某一阶段状态、但看起来像正式入口的文档
- 已经被新方案覆盖的临时说明
