## 1. 为什么这份文档不再作为入口

这份旧文档的问题是：

- 更偏原则讨论，不够像打包机上的执行手册
- 默认接受“安装包 + 模型包”的分包思路
- 没有把当前仓库的真实阻塞点绑定到具体文件

而轻量模式现在的产品口径已经明确：

- 面向用户应交付**一体化安装包**
- 不再要求用户额外下载或导入 `models_pack.zip`

因此，后续请改看：

- `docs/轻量模式/交付与Windows打包.md`

---

## 2. 新文档解决了什么问题

新文档已经补上：

- lite 一体化安装包的目标定义
- 当前仓库是否具备该目标的阻断检查
- Windows 打包机上的逐步执行顺序
- 构建后如何验收“是否真的内置了 lite 模型”
- 当前如果还打不出来，应如何向用户准确报阻塞

## 3. 这份旧文档保留用途
