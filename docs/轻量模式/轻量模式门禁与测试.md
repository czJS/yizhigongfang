# 轻量模式门禁与测试

> 本文合并了“发版门禁”“质量报告语义”“人工与自动化测试入口”。  
> 目标只有一个：判断当前这版轻量模式**能不能交付**。

---

## 1. 什么是轻量模式门禁

轻量模式门禁是一套“可交付底线 + 风险提示”规则，结果优先看每个任务目录的 `quality_report.json`，并结合轻量模式 smoke / regression / release 结果判断。

你最常看的字段是：

- `passed`
- `errors[]`
- `warnings[]`
- `checks{}`
- `metrics{}`

结论解释：

- `passed=false`：这条任务不应直接交付
- `passed=true`：底线通过，但仍可能存在风险项，需要结合 `warnings[]` 与少量人工抽检判断

---

## 2. 当前发版门禁

### 2.1 P0 必过

以下项不过，就不应进入交付：

1. Python `compileall` 通过
2. lite 门禁相关单测通过
3. 单条 lite 主路径 smoke 通过
4. `resume_from` smoke 通过
5. 代表样本 `golden20` smoke suite 通过
6. `quality_report.passed == true`
7. 最小交付件齐全：
   - `eng.srt`
   - `quality_report.json`
   - 字幕交付链：`output_en_sub.mp4`
8. 若本轮未启用 `skip_tts`：
   - `tts_plan.json`
   - `tts_full.wav`
   - `output_en.mp4`

### 2.2 P1 强烈建议

1. 字幕可读性抽检通过
2. 配音听感抽检通过
3. 成片遮挡抽检通过
4. 弱机器 / Windows 打包态资源闭环抽测通过
5. 失败提示与恢复路径抽检通过

---

## 3. `quality_report.json` 重点看什么

### 3.1 硬失败

当前最关键的硬失败包括：

- 必需产物缺失
- 英文字幕混入中文或全角
- 字幕时间轴非正时长
- 输出视频明显截断
- `resume_from` 缺失必要前置产物

### 3.2 告警

当前最重要的告警包括：

- 单行字幕过长
- 阅读速度过快
- 空字幕比例过高
- 时间轴重叠或过短
- TTS 预算风险
- 配音音频可能爆音
- 运行耗时相对源视频过高

---

## 4. 测试分层

### 4.1 人工测试重点

人工测试主要看这些用户可感知结果：

- 主路径能否跑通
- 听感是否自然
- 字幕是否可读
- 字幕位置是否遮挡关键画面
- 失败提示是否容易理解

更细的人工用例矩阵，看：

- `docs/轻量模式/轻量模式门禁与测试细化方案.md`
- `docs/轻量模式/人工测试用例.md`

### 4.2 自动化测试重点

自动化更适合覆盖：

- 代码与配置口径是否漂移
- 产物存在性
- `quality_report` 生成与语义
- `resume_from` 断点续跑
- 代表样本回归
- 运行耗时与超时诊断

自动化入口主要是：

- 统一脚本：`automation/scripts/lite_api_regress.py`
- 登录授权专项门禁：`automation/scripts/auth_api_regress.py`
- 开发轻量冒烟：`automation/configs/lite_dev_smoke.json`
- 日常自动化回归：`automation/configs/lite_regression_gate.json`
- 封版门禁：`automation/configs/lite_release_gate.json`
- 单条 smoke：`pipelines/tools/run_lite_pipeline_smoke_impl.py`
- 续跑 smoke：`pipelines/tools/run_lite_resume_smoke_impl.py`
- 代表样本 suite：`pipelines/tools/run_lite_golden20_smoke_suite_impl.py`

补充口径：

- 登录授权门禁已接入 `lite_api_regress.py`
- 因此运行 `lite_dev_smoke / lite_regression_gate / lite_release_gate` 时，会自动附带登录授权回归

更完整的自动化测试用例说明，看：

- `docs/轻量模式/自动化测试用例.md`

关系一定要分清：

- `自动化测试` 是总集合，指脚本、配置档和被脚本执行的检查项
- `门禁` 是自动化测试里的“带明确放行标准的预设组合”
- `封版门禁` 不是另一套系统，而是同一脚本里的最严格配置
- 因此可以理解为：`自动化测试 ⊇ 门禁`，`封版门禁` 只是其中最重的一档

## 5. 推荐测试顺序

### 5.1 静态与契约检查

1. Python `compileall`
2. lite 门禁相关单测
3. 如只改脚本编排或产物契约，先跑对应专项单测

### 5.2 轻量模式主链回归

建议按 3 层来跑：

1. 开发轻量冒烟：确认“代码没明显跑坏”
2. 日常自动化回归：确认“主链能力仍成立”
3. 封版门禁：确认“这版可以进入交付判断”

如果你只是改了单一高风险点，不建议一上来就跑重门禁，当前更推荐先跑专项：

1. 改产物契约、失败诊断、runner 逻辑：先跑 `test_run_lite_smoke_impl.py`、`test_run_lite_pipeline_smoke_impl.py`
2. 改 `resume_from`：先跑 `test_run_lite_resume_smoke_impl.py`
3. 改代表样本选择或套件统计：先跑 `test_run_lite_golden20_smoke_suite_impl.py`
4. 专项过了，再决定是否补 `dev-smoke` / `regression-gate`

推荐命令：

开发轻量冒烟：

```bash
python automation/scripts/lite_api_regress.py \
  --dev-smoke \
  --video "test_media/S4_dialog.mp4"
```

日常自动化回归：

```bash
python automation/scripts/lite_api_regress.py \
  --regression-gate \
  --video "test_media/S4_dialog.mp4"
```

封版门禁：

```bash
python automation/scripts/lite_api_regress.py \
  --release-gate \
  --video "test_media/S4_dialog.mp4"
```

三层的区别：

- `dev-smoke`：偏研发自测，先看主链有没有直接跑挂
- `regression-gate`：偏日常回归，覆盖主链 + 续跑 + 代表样本；默认用 `skip_tts=true` 验证字幕主链、续跑契约与产物契约
- `release-gate`：偏封版，覆盖最全，也最慢；默认验证完整 TTS 交付链
- 示例命令默认使用仓库内已存在的稳定样本 `test_media/S4_dialog.mp4`；若本轮要覆盖特定场景，也可替换为 `test_media/S1_clean.mp4`、`S2_hardsub.mp4`、`S3_fast.mp4`、`S5_noise.mp4` 或其他本地视频。

### 5.3 人工交付抽测

1. 主路径跑通
2. 检查输出件是否可直接拿给用户
3. 抽听配音、抽看字幕、抽看成片
4. 抽看失败提示是否可理解

## 6. 配置项测试的当前口径

当前轻量模式测试不再维护“大量历史开关逐项验收”的方式，而是收敛为：

- 一条默认主链
- 少量仍影响交付的功能项
- 统一轻量模式门禁

当前重点测试对象：

- 字幕更易读
- 续跑链路稳定
- 交付件齐全
- 时间轴与 TTS 协调
- 弱硬件下主路径稳定性

更细的自动化与人工测试用例矩阵，看：

- `docs/轻量模式/轻量模式门禁与测试细化方案.md`

---

## 7. 当前最小交付检查

发版前至少确认：

- 默认产品口径是单机本地运行
- 默认主链不依赖本地小模型或 Ollama
- 输出目录可见、可打开、可下载
- 缺关键资源、超时、缺前置产物时，失败信息可解释
- 若启用完整交付链，字幕、配音、成片输出目录中都能拿到
