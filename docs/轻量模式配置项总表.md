# 轻量模式配置项总表（全量版：含模型能力 + 全部参数 + 内部默认）

面向对象：**产品经理 / 交付负责人 / 测试 / 研发**（非技术同学也能读）  
范围：仅覆盖 **轻量模式（Lite Mode）**  
主入口：
- **Web UI/后端运行口径**：后端通过 `backend/task_manager.py::_build_cmd_lite` 组装命令并调用脚本 `scripts/asr_translate_tts.py`
- **直接命令行运行口径**：直接执行 `scripts/asr_translate_tts.py --help` 中的参数（与 Web UI 的“强制策略”可能不同）

> 重要口径（与质量模式总表一致）：
> - 本文的“配置项/可调项”按 **轻量模式实际生效的全量**统计：包括 UI/`defaults.yaml`/CLI 可改的项，也包括“代码里写死但会影响行为”的内部默认与阈值。
> - 同时会明确标注：**哪些键在 lite 模式并不生效（避免误用）**、哪些键被后端强制固定（例如 TTS 后端）。

## 1) 我采用的分类方式（与质量模式一致）

- **A 类：用户常用（建议在 UI 暴露）**  
  用户能理解、能自己决定，调错也不至于把系统“搞坏”。

- **B 类：高级设置（可在 UI 暴露，但要放到“高级”里）**  
  可能明显提升质量/体验，但也会带来明显成本/风险；需要说明与默认推荐。

- **C 类：工程/交付参数（不建议给普通用户；给交付/工程同学）**  
  多用于稳定性、离线部署、模型/路径/设备选择；需要工程理解。

- **D 类：内部固定默认（当前写死在代码里，但实际影响行为）**  
  不是“不能改”，而是“目前没有入口/不建议产品化”。本清单把它们**显式列出**，便于你决定后续是否要产品化。

- **E 类：调试/实验（不对外暴露）**  
  只为排障/研发/评测产生额外信息，不直接提升交付体验。

## 1.1 “是否暴露给用户”的定义（与质量模式一致）

- **UI 已暴露（可编辑）**：前端页面上有开关/输入框，用户改了会参与本次任务运行。
- **UI 已展示（只读/后端控制）**：前端显示但不可编辑（例如模型/设备/缓存目录），由后端配置/交付环境决定。
- **UI 未暴露**：前端没有入口，只能通过 `config/defaults.yaml`（或直接 CLI）配置。

> 备注：前端会过滤一批 **后端强制控制** 的键（`SERVER_CONTROLLED_KEYS`），即便前端显示，也不会提交覆盖后端配置。

## 1.2 参数类型与值类型（与质量模式一致）

- **参数类型**（怎么改）：开关 / 数值 / 枚举 / 文本 / 列表 / 门禁阈值 / 路径
- **值类型**（数据形态）：bool / int / float / enum / str / list / path

并给出：
- **默认值（defaults.yaml 当前设定）**：交付环境默认口径（最重要）
- **默认值（脚本 argparse 默认）**：代码兜底口径（用于理解/排障）
- **范围/可选值**：优先取 UI 的 `min/max/step` 或 CLI 的 `choices`；否则标 “-”

## 1.3 “实现来源”的定义（与质量模式一致）

- **代码实现（本项目能力）**：这是一套我们在流程里写了规则/步骤的能力（例如“句子单元”“自动专名保护”等）。
- **底层默认能力（模型/工具原生）**：底层引擎本来就支持的能力/参数（例如 whisper.cpp / Transformers / ffmpeg / TTS 引擎参数），本项目负责接入与封装。

---

## 1.4 “评测归类”（用于决定怎么测）

- **增强开关（单开关评测）**：典型 bool 开关；适合 A/B 对比，给出“默认开/高级/默认关”。
- **可调参数（扫参/灵敏度）**：数值/枚举；适合 short3 快筛 + golden9 复核。
- **工程回归**：路径/模型/设备/离线缓存等；更适合做稳定性与兼容性回归，不适合 onejob 式主评测。
- **环境/部署**：容器/磁盘/权限/架构等；更适合环境自检与说明文档。
- **门禁阈值**：影响质量报告判定标准；适合“阈值口径评审 + 回归测试”。

## 2) 轻量模式的“实际运行口径”说明（很关键）

- **2.1 Web UI/后端口径：轻量模式默认走 Piper（不在 UI 暴露）**  
  后端在 `backend/task_manager.py::_build_cmd_lite` 中会读取 `tts_backend`（默认来自 `config/defaults.yaml`），并将其传给脚本 `scripts/asr_translate_tts.py`：  
  - 默认：`tts_backend: piper`（轻量模式追求“完全离线 + 依赖最少”）  
  - 备注：脚本也支持 `coqui`，但当前前端不提供切换入口；如需切换，属于交付/工程配置范畴。

- **2.2 “VAD”在 Web UI 里可能看起来可选，但是否生效取决于是否配置了 VAD 模型文件**  
  whisper.cpp 的 VAD 需要 **额外的 `vad_model` 文件**。后端只有在找到该文件时，才会把 `--vad-enable/--vad-model` 传给脚本。  
  所以：**UI 上开了 VAD，不代表一定生效**（这不是前端 bug，是后端安全策略：没有模型就不启用）。

- **2.3 `dedupe` 当前属于“UI 有开关，但 lite 脚本不读取”的占位项**  
  lite 脚本内部始终会做一套“轻量去重/规整”规则，但并没有接 `--dedupe` 参数；因此该键在 lite 模式里**目前不生效**（建议后续要么接入，要么从 UI 去掉）。

### ASR（识别）

#### A 类（建议普通用户暴露）

- **`asr_normalize_enable`：识别文本净化（ASR 文本清洗）**
  - **一句话作用**：对识别出来的中文做“无损清洗”（空白/标点/不可见字符/常见错字替换），让后续翻译更稳定。
  - **用户价值（为什么要开）**：减少“脏字符/标点混乱/错别字”引起的误翻、漏翻、断句异常；字幕更规整。
  - **潜在风险（需要注意什么）**：若你配置了纠错字典且字典写错，可能误改专名；否则风险很低。
  - **适用场景（什么时候最有用）**：几乎所有素材；尤其口语多、识别错字多的素材。
  - **产品建议**：**建议默认开启**（低风险稳定性增强）。
  - **暴露是否符合分类**：符合
  - **前端入口（现状）**：UI已暴露(可编辑)
  - **参数形态**：开关（bool）
  - **评测归类**：增强开关（单开关评测）
  - **实现来源**：代码实现（本项目 ASR 文本净化逻辑 + 可选字典）
  - **默认（defaults.yaml）**：true
  - **默认（脚本）**：false（`--asr-normalize-enable` 为 store_true）
  - **CLI**：`--asr-normalize-enable`
  - **范围/可选**：-

- **`denoise`：识别前去噪**
  - **一句话作用**：在抽取音频时对噪声做一次“轻量去噪”，提高 whisper.cpp 在嘈杂素材上的稳定性。
  - **用户价值**：可能减少漏词/错词，改善字幕可读性（收益与素材相关）。
  - **潜在风险**：可能略增耗时；个别素材可能引入音色变化导致识别波动。
  - **适用场景**：环境噪声大、底噪明显、室外/人声混杂素材。
  - **产品建议**：建议放在普通用户区，默认按当前设定；遇到噪声素材再开。
  - **暴露是否符合分类**：符合
  - **前端入口（现状）**：UI已暴露(可编辑)
  - **参数形态**：开关（bool）
  - **评测归类**：增强开关（单开关评测）
  - **实现来源**：底层默认能力（ffmpeg 去噪滤镜；本项目负责启用与回退）
  - **默认（defaults.yaml）**：false
  - **默认（脚本）**：false（store_true）
  - **CLI**：`--denoise`
  - **范围/可选**：-

- **`sample_rate`：音频采样率（Hz）**
  - **一句话作用**：控制抽取音频与最终音频导出的采样率（影响识别输入与配音输出的基础质量/体积）。
  - **用户价值**：一般不需要改；少数素材/设备兼容时可调整。
  - **潜在风险**：采样率越高文件更大；过低可能影响识别效果。
  - **适用场景**：兼容性排障、需要与其他链路保持一致时。
  - **产品建议**：普通用户可隐藏；高级里保留（默认 16000 对 ASR 友好）。
  - **暴露是否符合分类**：基本符合（更偏 B/C）
  - **前端入口（现状）**：UI已暴露(可编辑)
  - **参数形态**：数值（int）
  - **评测归类**：工程回归
  - **实现来源**：底层默认能力（ffmpeg 音频抽取/重采样参数）
  - **默认（defaults.yaml）**：16000
  - **默认（脚本）**：16000

#### B 类（建议高级设置暴露）

  - **用户价值**：可用于微调分段与漏词风险。
  - **潜在风险**：调高可能漏词，调低可能切得太碎；且仅在 VAD 真正生效时有效。
  - **适用场景**：已启用 VAD 且发现“切太碎/漏词”需要平衡时。
  - **产品建议**：建议放高级，默认保守（0.6）。
  - **暴露是否符合分类**：偏 B
  - **前端入口（现状）**：UI已暴露(可编辑)
  - **参数形态**：数值（float）
  - **评测归类**：可调参数（扫参/灵敏度）

  - **一句话作用**：越大越不容易切分（段更长）。

  - **潜在风险**：过大可能把不该合并的句子合并到一起；且仅在 VAD 生效时有效。
  - **适用场景**：已启用 VAD 且切分过碎。
  - **产品建议**：建议放高级，默认 1.5。
  - **暴露是否符合分类**：偏 B

### MT（翻译与术语）

#### B 类（建议高级设置暴露）

- **`sentence_unit_enable`：句子单元（翻译前保守合并）**
  - **一句话作用**：翻译前把相邻短句“保守合并”成更完整的句子，再按原时间轴拆回，提升上下文与语法完整性。
  - **用户价值**：有机会减少碎句直译导致的生硬、指代漂移。
  - **潜在风险**：合并/拆分会引入轻微错位风险（尤其时间轴断点处）；对某些素材可能反而变差。
  - **适用场景**：口语碎片多、短句密集、翻译容易断裂的素材。
  - **产品建议**：建议放高级，默认关闭；需要时开启并配合抽样复核。
  - **暴露是否符合分类**：符合（B 类）
  - **前端入口（现状）**：UI未暴露（当前仅质量模式面板有入口；lite 模式默认看不到）
  - **参数形态**：开关（bool）
  - **评测归类**：增强开关（单开关评测）
  - **实现来源**：代码实现（本项目句子单元合并/回填）
  - **默认（defaults.yaml）**：false
  - **默认（脚本）**：false（store_true）
  - **CLI**：`--sentence-unit-enable`
  - **范围/可选**：-

- **`sentence_unit_min_chars`：句子单元最小字数**
  - **一句话作用**：控制“合并到多长才算有足够上下文可闭合”。
  - **用户价值**：平衡“上下文”与“错位风险”。
  - **潜在风险**：过大可能合并过度；过小可能提升不明显。
  - **适用场景**：已开启句子单元，需要微调风格。
  - **产品建议**：高级项；默认 12。
  - **前端入口（现状）**：UI未暴露
  - **参数形态**：数值（int）
  - **评测归类**：可调参数（扫参/灵敏度）
  - **实现来源**：代码实现
  - **默认（defaults.yaml）**：12
  - **默认（脚本）**：12
  - **CLI**：`--sentence-unit-min-chars`
  - **范围/可选**：-

- **`sentence_unit_max_chars`：句子单元最大字数**
  - **一句话作用**：限制合并后的最大长度，避免过长导致翻译与回填风险上升。
  - **产品建议**：高级项；默认 60。
  - **前端入口（现状）**：UI未暴露
  - **参数形态**：数值（int）
  - **评测归类**：可调参数（扫参/灵敏度）
  - **实现来源**：代码实现
  - **默认（defaults.yaml）**：60
  - **默认（脚本）**：60
  - **CLI**：`--sentence-unit-max-chars`
  - **范围/可选**：-

- **`sentence_unit_max_segs`：句子单元最多合并段数**
  - **一句话作用**：限制一次合并跨越的 ASR 段数，降低错位风险。
  - **产品建议**：高级项；默认 3。
  - **前端入口（现状）**：UI未暴露
  - **参数形态**：数值（int）
  - **评测归类**：可调参数（扫参/灵敏度）
  - **实现来源**：代码实现
  - **默认（defaults.yaml）**：3
  - **默认（脚本）**：3
  - **CLI**：`--sentence-unit-max-segs`
  - **范围/可选**：-

- **`sentence_unit_max_gap_s`：句子单元最大跨段间隔（秒）**
  - **一句话作用**：跨段静音间隔太大则不合并，避免跨场景硬合并。
  - **产品建议**：高级项；默认 0.6。
  - **前端入口（现状）**：UI未暴露
  - **参数形态**：数值（float）
  - **评测归类**：可调参数（扫参/灵敏度）
  - **实现来源**：代码实现
  - **默认（defaults.yaml）**：0.6
  - **默认（脚本）**：0.6
  - **CLI**：`--sentence-unit-max-gap-s`
  - **范围/可选**：-

- **`sentence_unit_boundary_punct`：句子单元闭合标点**
  - **一句话作用**：当句尾命中这些标点且上下文足够时，允许结束合并单元。
  - **产品建议**：高级项；默认 `。！？!?.,`。
  - **前端入口（现状）**：UI未暴露
  - **参数形态**：文本（str）
  - **评测归类**：工程回归
  - **实现来源**：代码实现
  - **默认（defaults.yaml）**：`"。！？!?.,"`
  - **默认（脚本）**：`"。！？!?.,"`
  - **CLI**：`--sentence-unit-boundary-punct`
  - **范围/可选**：-

- **`sentence_unit_break_words`：句子单元断点词（逗号分隔或列表）**
  - **一句话作用**：遇到“但/而/于是/然后…”这类明显转折/并列词，更倾向断开，减少硬合并错译。
  - **产品建议**：高级项；默认见 `defaults.yaml` 列表。
  - **前端入口（现状）**：UI未暴露
  - **参数形态**：列表（list[str]）或 文本（str）
  - **评测归类**：工程回归
  - **实现来源**：代码实现
  - **默认（defaults.yaml）**：`["但","而","于是","然后","忽然","突然","不过","结果","同时"]`
  - **默认（脚本）**：`""`（空字符串）
  - **CLI**：`--sentence-unit-break-words`
  - **范围/可选**：-

- **`entity_protect_enable`：自动专名/术语保护**
  - **一句话作用**：从中文字幕中“保守抽取”专名候选→先单独翻译得到稳定英文→用占位符保护→翻译后回填，提升译名一致性与配音可读性。
  - **用户价值**：减少专名漂移、同一名词多译；对剧情/人物/地名密集素材更有价值。
  - **潜在风险**：候选抽取过少则收益有限；候选抽取误伤会引入不自然英文（但本项目已做极保守策略）。
  - **适用场景**：人物/地名/专有名词密集，且你对译名一致性要求高。
  - **产品建议**：建议放高级，默认关闭；开启后建议抽样检查“保护的词是否合理”。
  - **前端入口（现状）**：UI未暴露（当前仅质量模式面板有入口；lite 默认看不到）
  - **参数形态**：开关（bool）
  - **评测归类**：增强开关（单开关评测）
  - **实现来源**：代码实现（本项目自动候选抽取 + 占位符保护）
  - **默认（defaults.yaml）**：false
  - **默认（脚本）**：false（store_true）
  - **CLI**：`--entity-protect-enable`
  - **范围/可选**：-

- **`entity_protect_min_len / entity_protect_max_len / entity_protect_min_freq / entity_protect_max_items`：专名候选抽取阈值**
  - **一句话作用**：控制“哪些中文片段可能被当作专名候选”。
  - **用户价值**：在“候选太少/太多”时可调。
  - **潜在风险**：本项目内部有硬限制（更保守）：`min_freq>=4`、`max_items<=8`，因此扫参空间有限。
  - **产品建议**：更偏工程/高级；默认按 `defaults.yaml` 即可。
  - **前端入口（现状）**：UI未暴露
  - **参数形态**：数值（int）
  - **评测归类**：可调参数（扫参/灵敏度）
  - **实现来源**：代码实现（本项目候选抽取策略）
  - **默认（defaults.yaml）**：`min_len=2, max_len=8, min_freq=4, max_items=8`
  - **默认（脚本）**：`2/6/2/30`（但会被内部硬限制再收紧）
  - **CLI**：`--entity-protect-min-len/--entity-protect-max-len/--entity-protect-min-freq/--entity-protect-max-items`
  - **范围/可选**：-

  - **一句话作用**：指定离线翻译模型（支持 HuggingFace 模型 ID 或本地目录路径）。
  - **用户价值**：决定翻译质量/速度/资源占用，是“硬件档位”的核心。
  - **潜在风险**：离线模式下模型缺失会直接报错；大模型更吃内存。
  - **适用场景**：交付部署、硬件分档（normal/mid/high）。
  - **产品建议**：前端只读展示；由预设/交付环境决定。
  - **前端入口（现状）**：UI已展示(只读/后端控制)
  - **参数形态**：文本（str）
  - **评测归类**：工程回归
  - **实现来源**：底层默认能力（Transformers 模型）
  - **默认（defaults.yaml）**：见 presets（例如 `presets.normal.mt_model`）
  - **默认（脚本）**：`Helsinki-NLP/opus-mt-zh-en`
  - **CLI**：`--mt-model`
  - **范围/可选**：-

  - **一句话作用**：控制翻译模型用 CPU 还是 GPU（auto 为自动）。
  - **产品建议**：前端只读展示；由交付环境决定。

  - **范围/可选**：auto/cpu/cuda

  - **一句话作用**：禁止 Transformers/HF 在运行期联网下载模型，确保可控与可复现。
  - **用户价值**：交付可控；避免“线上突然下载失败导致任务失败”。

  - **适用场景**：离线交付、生产环境。

  - **前端入口（现状）**：UI未暴露
  - **参数形态**：开关（bool）
  - **评测归类**：环境/部署
  - **实现来源**：底层默认能力（Transformers local_files_only）
  - **默认（defaults.yaml）**：true

  - **一句话作用**：当 `sentence_unit_enable=false` 时，脚本仍会用一套历史保守策略做小范围合并（最多 2 段 / 或碰到标点 / 或累计字数阈值）。

- **内部默认：翻译输出的轻量规整与去重**
  - **一句话作用**：脚本内部始终会做：空白规整、常见短语替换、简单 n-gram 去重、短语窗口去重，降低“重复长句导致 TTS 崩坏”的概率。
  - **是否值得暴露**：建议固定（D 类）；若要产品化，应新增明确开关并配套评测。
  - **评测归类**：工程回归
  - **实现来源**：代码实现（脚本内部 `rule_polish/dedupe_repeats/dedupe_phrases`）

### TTS（配音）

#### A 类（建议普通用户暴露）

- **`tts_split_len`：TTS 单段最长字符（拆分阈值）**
  - **一句话作用**：英文句子过长时先拆分再合成，降低 TTS “重复/飙速/卡死”的概率。
  - **用户价值**：提升稳定性与听感一致性。
  - **潜在风险**：拆得太短可能更碎；太长可能增加重复风险。
  - **适用场景**：任何素材；尤其长句多的解说类。
  - **产品建议**：可放高级（默认 80 足够稳）。
  - **前端入口（现状）**：UI未暴露
  - **参数形态**：数值（int）
  - **评测归类**：可调参数（扫参/灵敏度）
  - **实现来源**：代码实现（本项目 TTS 拆分与对齐策略）
  - **默认（defaults.yaml）**：80
  - **默认（脚本）**：80
  - **CLI**：`--tts-split-len`
  - **范围/可选**：-

- **`tts_speed_max`：TTS 最大允许提速倍数**
  - **一句话作用**：当合成语音比时间轴更长时，允许最多提速到多少来对齐，避免无限提速导致“听感崩坏”。
  - **用户价值**：在“文本长/时长短”的段落里更稳，更少被硬截断。
  - **潜在风险**：值越大越可能出现“飙语速”听感问题；值越小越可能被截断或补静音。
  - **适用场景**：追求更自然听感时可适当降低；追求对齐时可适当提高。
  - **产品建议**：建议作为高级可调；默认 1.1（相对保守）。
  - **前端入口（现状）**：UI未暴露
  - **参数形态**：数值（float）
  - **评测归类**：可调参数（扫参/灵敏度）
  - **实现来源**：代码实现（本项目对齐策略 + 速度上限）
  - **默认（defaults.yaml）**：1.1
  - **默认（脚本）**：1.1
  - **CLI**：`--tts-speed-max`
  - **范围/可选**：-

  - **范围/可选**：-

- **`min_sub_duration`：最短字幕时长（秒）**
  - **一句话作用**：把过短字幕段适当延长，减少“闪字幕/难以阅读”与配音对齐压力。
  - **用户价值**：可读性更稳、TTS 更不容易超时。
  - **潜在风险**：会改变时间轴末端（延长 end）；极端情况下可能影响后续对齐精细度。
  - **产品建议**：建议放高级；默认 1.5。
  - **前端入口（现状）**：UI未暴露
  - **参数形态**：数值（float）
  - **评测归类**：可调参数（扫参/灵敏度）
  - **实现来源**：代码实现（本项目字幕段最短时长补偿）
  - **默认（defaults.yaml）**：1.5
  - **默认（脚本）**：1.5
  - **CLI**：`--min-sub-dur`
  - **范围/可选**：-

- **`skip_tts`：跳过配音（只产出字幕）**
  - **一句话作用**：只跑 ASR+翻译输出字幕，不跑配音与成片复合。
  - **用户价值**：只要字幕交付时大幅省时。
  - **潜在风险**：不会生成配音与成片；属于预期。
  - **产品建议**：可作为高级开关。
  - **前端入口（现状）**：UI未暴露
  - **参数形态**：开关（bool）
  - **评测归类**：工程回归
  - **实现来源**：代码实现（本项目流程裁剪）
  - **默认（defaults.yaml）**：未配置
  - **默认（脚本）**：false（store_true）
  - **CLI**：`--skip-tts`
  - **范围/可选**：-

### 质量门禁阈值（共享能力：lite/quality/online 通用）

> 这些阈值来自 `config/defaults.yaml: quality_gates`，影响 `quality_report.json` 的判定与前端红标提示。

- **`quality_gates.allow_cjk_in_english_srt`：英文字幕是否允许出现中文（硬门禁）**
  - **一句话作用**：英文字幕中出现中文/全角字符是否直接判失败。
  - **默认（defaults.yaml）**：false

- **`quality_gates.max_chars_per_line`：英文单行最大字符数（告警门禁）**
  - **默认（defaults.yaml）**：80

- **`quality_gates.max_cps`：英文字幕最大阅读速度 CPS（告警门禁）**
  - **默认（defaults.yaml）**：20.0

- **`quality_gates.max_empty_ratio`：空行率上限（告警门禁）**
  - **默认（defaults.yaml）**：0.10

- **`quality_gates.max_truncation_s / max_truncation_ratio`：成片截断门禁（硬失败）**
  - **默认（defaults.yaml）**：`1.0 / 0.03`

## 附录 C：`defaults.yaml` 中存在但轻量模式（Web UI/后端）不生效的键（避免误用）

轻量模式脚本是 `scripts/asr_translate_tts.py`。以下这些能力/键主要属于 **质量模式脚本 `scripts/quality_pipeline.py`**（或在线模式），在 lite 模式里：
- **不会被后端传参**（即便 defaults.yaml 有默认值）
- 或 **脚本不识别该 CLI 参数**

常见的“质量模式专用”前缀（不在此文逐一展开，详见 `docs/质量模式配置项总表.md`）：
- `whisperx_*`, `llm_*`
- `tra_*`, `qe_*`
- `subtitle_*`（字幕后处理/回流）、`display_*`（展示版字幕）
- `tts_fit_*`, `tts_plan_*`
- `bgm_*`（混音）、`erase_subtitle_*`（擦字幕）
- `glossary_placeholder_*`, `glossary_prompt_enable`

> 备注：如果你希望把其中某些“质量增强”能力引入轻量模式，需要同时满足：
> - lite 脚本支持对应参数/逻辑（或后端改为调用 quality 脚本）
> - 评测流程与默认策略重新对齐（避免“lite 变成另一个 quality”）
