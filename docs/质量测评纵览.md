## 0) 这份文档怎么读（给非技术）

- **先看通过率 `passed_rate`**：掉通过率意味着更容易“翻车”，一般不建议默认开。  
- **再看 E2E 体验分 `e2e_mean / e2e_score_100`**：这是“可交付体验”的工程评分（0–100）。  
- **最后看 `p_improve`（置信度）**：越接近 1 越可信；接近 0 表示“多数抽样里不比基线好”。  
- **金标分（ASR/MT）用于解释原因**：当 E2E 变化不明显时，用金标分判断是“真提质”还是“只是工程规则变化”。

## 1) 本轮评测对象口径：什么算“增强开关”

### 1.1 “增强开关”是什么？

这里的增强开关指：**开 / 关**就会改变端到端交付体验或质量的布尔能力（例如：TRA、QE、字幕后处理、TTS fit 等）。  
仓库当前采用的“全量清单口径”为 **46 个单开关**（见手册第 4/5 节）。

### 1.2 46 个开关里，为什么有些“不测”？

手册里明确了“本轮不测”的几类原因（这不是偷懒，而是为了让结论更可复用、更可决策）：

- **默认开启（不测）**  
  - 例：`asr_normalize_enable`（ASR 文本净化）在 `config/quality.yaml` 已默认开，属于“基线的一部分”，不再作为单开关实验项重复测。
- **剔除：Debug**（只写调试文件/报告，不应当影响体验分）  
  - 例：`asr_merge_save_debug`、`asr_llm_fix_save_debug`、`meaning_split_save_debug`、`tts_fit_save_raw`、`tra_save_debug`、`qe_save_report`
- **剔除：对比辅助 / 工作流**（不是增强能力，结论不可比或需要人工介入）  
  - 例：`mt_skip_if_present`（复用旧翻译）、`mt_pause_before_translate`（译前暂停人工编辑）、`skip_tts`（跳过配音，不算完整交付）
- **剔除：已下线/风险高（当前不生效或回归风险大）**  
  - 例：`llm_selfcheck_enable`
- **剔除：占位/未实现**（传参存在，但链路不产生实际效果）  
  - 例：`diarization`
- **剔除：字典/数据驱动**（结论强依赖外部 glossary/字典内容，难以泛化）  
  - 例：`glossary_placeholder_enable`、`glossary_prompt_enable`
- **剔除：审计项**（更像“查问题”而不是“自动提升交付体验”）  
  - 例：`qe_backtranslate_enable`

> 以上“剔除/默认开不测”的归因与口径，均来自 `eval/QUALITY_SWITCH_TEST_ROUND1_MANUAL.md`。

## 3) Round1 结论总览（已跑结果，含中文注释）

> Round1 的核心目标：用 golden9/short3 跑“单开关对比”，得出**默认开 / 高级开 / 默认关（或隐藏）**的产品决策建议。

### 3.1 P0（单开关：产品最关心的一组）

事实来源：`reports/e2e_quality/report_golden9_round1_master.md`

#### 3.1.1 结论（可直接用于评审）

- **推荐默认开启（Recommended ON）**
  - **`subtitle_postprocess_enable`（字幕后处理：软换行 + CPS 修正）**  
    - 证据：`e2e_mean=85.47`，相对 baseline **Δe2e=+2.47**，且 **`p_improve(e2e)=1.0`**，`passed_rate=1.0`  
    - 白话解释：字幕更“好读”、更不容易触发可读性门禁，且稳定交付不受影响。

- **建议作为高级选项（Advanced / On-demand）**
  - **`tra_enable`（TRA 多步翻译）**  
    - 证据：Δe2e=+0.60，`p_improve(e2e)=0.991`（体验侧很可信）  
    - 风险/成本：本轮 `MT(final100)` 反而下降（Δ=-0.69），且链路更慢、更依赖 LLM 稳定性  
    - 产品建议：更适合“追求更自然口语、接受更慢/更贵”的用户
  - **`sentence_unit_enable`（句子单元合并）**  
    - 证据：E2E 基本不变，但 `MT(final100)` 提升明显（Δ=+0.52，`p_improve(MT)=1.0`）  
    - 产品建议：更像“翻译一致性/语义连贯性”的高级旋钮，不建议默认开。
  - **`qe_enable`（QE 质量评审 + 选择性修复）**  
    - 证据：E2E 无提升，但 `MT(final100)` 有小幅提升且置信度高（Δ=+0.08，`p_improve(MT)=0.976`）  
    - 产品建议：适合“术语/专名/数字多、对错译更敏感”的场景按需开。
  - **`tts_fit_enable`（TTS 超时裁剪兜底）**、**`tts_plan_enable`（TTS 语速规划/停顿审计）**  
    - 证据：E2E 变化小（或中等置信），更像“风险兜底/交付稳定性工具”  
    - 产品建议：先做高级选项；后续用“更长视频/更密集字幕样本”复测其兜底收益是否能稳定出现。

- **不建议默认开（建议隐藏或仅内部实验）**
  - **`tts_script_enable`（字幕稿/朗读稿分离）**  
    - 证据：`e2e_mean=71.35`，Δe2e=-11.65，`p_improve(e2e)=0.0`（明确负收益）  
    - 白话解释：当前策略会显著伤害交付体验分（即使 MT 分未必变差）。  
    - 产品建议：默认关并隐藏；若要继续探索，建议单独优化朗读稿生成策略再复测。
