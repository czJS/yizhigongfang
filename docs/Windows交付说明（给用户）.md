# Windows 交付说明（给用户）

> 适用：你拿到的是 **Windows 安装包 `YizhiStudio-*.win.exe`** + **模型包 `models_pack.zip`** +（可选）**Ollama 包 `ollama_pack.zip`**  
> 目标：安装后无需 Python，可直接使用；质量模式需要额外导入 Ollama 包。

---

## 你会拿到哪些文件

- **安装包**：`YizhiStudio-*.win.exe`
- **模型包**：`models_pack.zip`
- **Ollama 包（质量模式用）**：`ollama_pack.zip`（如果交付方提供）

> 注意：安装包默认不内置模型（体积太大）。模型需要在 App 内导入一次。

---

## 1. 安装

1) 双击运行 `YizhiStudio-*.win.exe` 完成安装  
2) 安装完成后打开 **YizhiStudio**

---

## 2. 导入模型包（必须做一次）

1) 打开 App  
2) 进入“模型/资源/离线资源”（或类似入口）  
3) 点击“选择模型包 / 导入模型包”  
4) 选择你拿到的 `models_pack.zip`  
5) 等待导入完成（可能需要几分钟到几十分钟，取决于硬盘与包大小）

### 模型会被解压到哪里？

App 会把模型解压到用户数据目录下：

- 优先：**安装目录旁边**的 `user_data\models\`
- 其次：`D:\dubbing-gui\...`（如果存在）
- 否则：Windows 默认用户目录（不建议）

---

## 3. 验收：确认后端与质量模式可用

### 3.1 后端健康检查

打开浏览器访问：

- `http://127.0.0.1:5175/api/health`

看到 `ok` / `status: ok` 之类即表示后端已正常启动。

### 3.2 质量模式（LLM）检查

质量模式依赖本机 Ollama 服务（App 会尝试自动启动）。如果交付方给了 `ollama_pack.zip`，请先导入它（见下一节）。

---

## 3.3 导入 Ollama 包（质量模式需要）

1) 打开 App  
2) 进入 **系统页 → Ollama 管理**  
3) 点击“选择 Ollama 包并导入”  
4) 选择你拿到的 `ollama_pack.zip`  
5) 导入完成后，等待几秒或重启 App（让 App 自动拉起 Ollama 服务）

---

## 4. 常见问题与处理

### 4.1 提示“未检测到模型/缺模型”

- 你还没导入 `models_pack.zip`，或导入失败
- 重新在 App 内执行“导入模型包”
- 确认磁盘空间足够（建议预留 50GB+，视模型而定）

### 4.2 质量模式提示 LLM 连接失败 / 找不到 Ollama

排查顺序：

1) **先导入 `ollama_pack.zip`**（系统页 → Ollama 管理）
2) **确认模型包里包含 LLM 模型**（有些交付会把 `ollama_models` 放进 `models_pack.zip`）
3) 关闭并重启 App（让 App 重新拉起 Ollama）
4) 确认本机 11434 端口未被占用（如被占用，关闭占用程序后重启）

### 4.3 导入模型包很慢/失败

- 尽量把 `models_pack.zip` 放到本机 SSD 上再导入
- 如果你的电脑装了杀毒/管家软件，可能会拖慢大量小文件写入
- 若提示 Zip 解压失败，安装 7-Zip（可提升大包解压稳定性）
